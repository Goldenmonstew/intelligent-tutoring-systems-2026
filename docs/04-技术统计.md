# 技术统计：方向 · 方法 · 成熟度

## 方法

- **对象**：官方程序全 **71 篇 census**（非抽样），逐篇编码。**完整编码表见 [`../data/coding.csv`](../data/coding.csv)**——内含三套编码列（`*_content`、`*_title_a`、`*_title_b`）与共识列（`*_final`），可据此复算下文全部比例与一致性。
- **三套编码口径**（互不参照、统一编码手册）：
  - **内容编码（content）**：结合 14 场录像的报告内容编码（信息量最高，对"是否用到 LLM"判断最准）。
  - **标题编码 ①、②（title_a / title_b）**：两套各自独立、**仅依据论文标题**的编码，且彼此盲编。
  - 说明：本文报告的 κ 是**编码轮之间（pass-level）的一致性**，不等同于多名独立人类编码者间的信度。
- **共识与裁决规则**：
  - 主轨道、agentic：取三套**多数**；若 1-1-1 三者皆异（或缺内容编码且两套标题不一致），归**内容编码**值（缺内容时归标题①）。
  - LLM 用法：**以内容编码为准**（"仅标题"口径会系统性低估，见下）。
  - 缺失处理：71 篇中约 60 篇有可用口头报告内容；缺席/跳过/仅报题的论文无内容编码，按标题口径补码。主轨道与 agentic 覆盖全 71 篇；LLM 涉及率按 71 篇计（缺席论文以标题判定，倾向保守）。

## 操作定义（编码手册）

- **主轨道（单选）**：取论文**主要贡献**所属轨道；跨轨论文归于"研究问题 / 方法核心"所在轨道。
- **LLM 用法**：`核心`=LLM 输出直接驱动教学反馈/诊断/评分/核心建模；`外围`=LLM 仅用于数据合成/标注/转录/润色/界面；`无`=未使用。
- **Agentic（是）**：显式多智能体/智能体编排架构（多分工角色 + 协调/编排），或论文自述为 agentic 框架；单个对话式智能体不计。

## 编码轮一致性与跨信息源差异（n=71）

> **标题① vs 标题②** 看同一份材料（标题），衡量编码可复现性；**内容 vs 标题** 看不同信息源，衡量口径敏感性（差异里混入信息量差别，不应直接当作编码不可靠）。下方"内容 vs 标题"的"标题"指标题①。

| 维度 | 标题① vs 标题②（同源）| 内容 vs 标题①（跨信息源）| 三套全一致 |
|---|---|---|---|
| 是否智能体架构 | κ=1.000 | κ=0.882 | 95.8% |
| 主轨道 | κ=0.858 | κ=0.733 | 74.6% |
| LLM 用法 | κ=0.932 | κ≈0.33 | 56.3% |

**关于 LLM 维度**：两套独立的"仅标题"编码彼此 κ=0.932（标题编码本身高度可复现），却都同向偏离结合报告内容的编码——这**较强地提示：仅依赖标题会系统性低估 LLM 使用，该偏差具有方向稳定性，而非仅随机波动**。标题显式提及 LLM 的仅约 28–30%，但结合报告内容，**71 篇中 49 篇（69%）涉及 LLM**。

## 几点订正（相对粗略的量级估计）

1. **T1+T7 = 36/71 ≈ 51%**，低于"约三分之二"的直觉印象。
2. **T4（学习分析/知识追踪）是第二大主轨（18/71，25%）**，高于 T7（13/71，18%）。
3. **LLM 实际涉及 49/71（69%，核心 37/71=52%）**，介于"标题显式 28%"与"无处不在"之间。
4. **智能体架构 17/71（24%）**，是一致性最高的维度（标题两套 κ=1.0、三套 95.8% 一致）。

## 成熟度（单轮估计，低置信）

> 该维度依据报告内容单轮估计，未做多套口径交叉编码，置信度低于上面三个维度。

- 纯概念 / 框架、无实证：约 15%
- 试点 / 小样本（n≈5–40）/ 弱对照：约 55–60%（多数）
- 较规范评估（较大公开数据集 / 对照 / 统计检验）：约 25%

> "过半为试点 / 小样本"的方向性判断较稳；精确比例待进一步核验。
