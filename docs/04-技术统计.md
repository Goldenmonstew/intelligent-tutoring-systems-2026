# 技术统计：方向 · 方法 · 成熟度

## 方法

- **对象**：官方程序全 **71 篇 census**（非抽样），逐篇编码。**完整编码表见 [`../data/coding.csv`](../data/coding.csv)，可据此复算下文全部比例。**
- **三套编码口径**（互不参照、统一编码手册）：
  - **内容编码**：结合 14 场录像的报告内容编码（信息量最高，对"是否用到 LLM"判断最准）。
  - **标题编码 ①、②**：两套各自独立、**仅依据论文标题**的编码。
- **缺失处理**：71 篇中约 60 篇有可用口头报告内容，其余（缺席/跳过/仅报题）按标题口径补码。主轨道与 agentic 维度覆盖全 71 篇；LLM 实际涉及率按 71 篇计（缺席论文以标题判定，倾向保守，故可能略偏低）。

## 操作定义（编码手册）

- **主轨道（单选）**：取论文**主要贡献**所属轨道；跨轨论文归于"研究问题 / 方法核心"所在轨道。例如"用 LLM 做 SQL 辅导"归 **T1 智能辅导**（贡献是辅导系统），而非 T7。
- **LLM 用法**：
  - `核心`：LLM 输出直接驱动教学反馈、诊断、评分或核心建模；
  - `外围`：LLM 仅用于数据合成、标注、转录、润色或界面包裹；
  - `无`：未使用 LLM / 生成式 AI。
- **Agentic（是）**：显式采用多智能体 / 智能体编排架构（多个分工角色 + 协调/编排），或论文自述为 agentic 框架；单个对话式智能体不计。

## 一致性与跨信息源差异

> 区分两类比较：**标题① vs 标题②** 看的是同一份材料（标题），衡量编码的可复现性；**内容 vs 标题** 看的是不同信息源，衡量口径敏感性，差异里混入了信息量差别，不应直接当作编码不可靠。

| 维度 | 标题① vs 标题②（同源双编码）| 内容 vs 标题（跨信息源）| 三套全一致 |
|---|---|---|---|
| **是否智能体架构** | κ=1.000 | κ=0.882 | 95.8% |
| **主轨道** | κ=0.858 | κ=0.733 | 74.6% |
| **LLM 用法** | κ=0.932 | κ≈0.33 | 56.3% |

**关键观察（LLM 维度）**：两套完全独立的"仅标题"编码彼此 κ=0.932（近乎完全一致，说明标题编码本身很可复现），却都同向偏离结合报告内容的编码——这**强烈表明"只看标题会系统性低估 LLM 使用"是稳定的系统偏差，而非随机噪声**。标题显式提到 LLM 的仅约 28–30%，但结合报告内容，**71 篇中约 49 篇（69%）涉及 LLM**。

## 几点订正（相对粗略的量级估计）

1. **T1+T7 ≈ 51%（23+13=36/71）**，低于"约三分之二"的直觉印象。
2. **T4（学习分析/知识追踪）是第二大主轨（25%，18/71）**，高于 T7（18%，13/71）；容易被并入"约 10%"而低估。
3. **LLM 实际涉及约 69%（49/71，核心 52%）**，介于"标题显式 28%"与"无处不在"之间；标题口径的低估已被两套独立标题编码 κ=0.93 印证。
4. **智能体架构约 24%（17/71）**，是一致性最高的维度（标题两套 κ=1.0、三套 95.8% 一致）。

## 成熟度（单轮估计，低置信）

> 该维度依据报告内容单轮估计，未做多套口径交叉编码，置信度低于上面三个维度。

- 纯概念 / 框架、无实证：约 15%
- 试点 / 小样本（n≈5–40）/ 弱对照：约 55–60%（多数）
- 较规范评估（较大公开数据集 / 对照 / 统计检验）：约 25%

> "过半为试点 / 小样本"的方向性判断较稳；精确比例待进一步核验。
