## 一、研究热点（Top 10）

> 提醒：本届主题即 *Agentic Learning Ecosystems*，会系统性抬高智能体类工作的可见度；下列排序解读"本届显性信号"。

1. **智能体 / 多智能体架构**（本届最突出的新增架构热点）— 约四分之一的报告显式采用；从单智能体走向编排器 / 陪审团 / 协调器，并进入蛋白折叠、驾驶舱辅助、教材审计等非传统场景。会上出现若干**互操作与编排机制**：数据标准（如 FHIR）、上下文/工具协议（如 MCP）、编排框架（如 LangGraph）、经典多智能体中间件（如 JADE/FIPA-ACL）——但它们分属不同层级，谈不上"统一标准栈已落地"。
2. **LLM 可靠性工程** — 幻觉检测（#30，其设置下 precision≈73%）、用编译器当裁判生成数据（#34）、LLM 当评委/陪审团（#4/#52）、token 概率置信度（#11）。
3. **"何时介入 / 克制自主" + 对抗过度依赖** — 注意：提示时机、脚手架渐隐、求助行为在文献里早非新问题；**新的是在 LLM 语境下与过度依赖耦合后被重新放大**（生产性摩擦、徘徊检测、介入分级）。
4. **知识追踪的工程化与细粒度化** — 分布式训练、元认知感知、置信度感知、可复现性再评估。
5. **可信 / 治理 AI** — 确定性运行时治理（#42）、公众信任调查（#41，N=618）、定性分析透明度规范（#45）。
6. **轻量化 / 本地化 / 低成本部署**（值得注意的次级实用主义线索）— 离线量化 LLM + RAG（#84）、任务特定下 7B+LoRA 接近大模型（#90）、边缘小模型（#55）。
7. **多模态情感 / 认知感知** — 眼动 + 鼠标、语音情感、用热成像替代 EEG。
8. **VR / 元宇宙 + 生成式 avatar** — 对话式引导、声景、avatar 教学；共性新问题"学生不知道该问什么"。
9. **经典学习理论写进架构**（设计范式，非自动等于更好效果）— 把最近发展区/心流/论证理论写进损失函数或设计决策。
10. **领域特化辅导** — SQL、阿拉伯手写、量子计算、营养、ESL 文本简化。

## 二、技术难点（按类归并）

**A. LLM 固有缺陷** — 幻觉（教育场景危害大；#30 在其设置下最佳幻觉检测 precision≈73%，不应据此解释为领域上限）；数值/算数不可靠（转而外包给计算器）；**提示复杂度须匹配模型规模**（思维链在小模型上反而过度思考、虚构）；上下文窗口上限；小模型在简单防护下更易被提示注入。

**B. 评估方法学（本届被反复检视的薄弱环节）** — 度量与学习结果脱节（"相同 F1 不同反馈"）；缺人工金标；**时间/前瞻泄漏致 AUC 虚高（#47，其数据上 0.72→0.96）**；"带工具会做"≠"学会了"（需迁移评估）；小样本/无对照/合成数据的乐观偏差；更深一层是**构念效度**（学到什么、迁移到哪、延迟保持、群体是否等效），而不只是 F1/AUC 失真。

**C. 自适应核心** — 何时介入；最近发展区的实时估计；脚手架渐隐的细粒度校准；过度依赖；**静态数据集到真实学生的迁移鸿沟**（离线高分 ≠ 真实课堂）。

**D. 数据** — 标注稀缺/冷启动；长尾稀有类；教育源码数据集质量差（#71，某 Java 集 <2% 可编译）；无标注情感数据（用代理/演员语音替代，个体声学不一致）。

**E. 多模态/情感** — 福祉/情感构念难自动检测；EEG 在某些场景不可行（改用热成像间接测）；单模态局限。

**F. 部署** — 延迟 vs 质量硬权衡（#43，最高质量约需 14 秒，为保心流牺牲推理换 <1 秒/约 80% 质量）；成本/能耗；**量化效果高度依赖模型/框架/显存/硬件/任务**（不可外推）；消费级 GPU 基准不可从数据中心外推。

**G. 信任 / 公平 / 治理** — 信任宜按连续量建模；拟人化在特定实验条件下与社会偏见相关；公平的资源约束当前缺稳健解；去偏见有副作用（可能牺牲反馈的同理心语气）。

**H. 工程 / 可复现** — 多智能体架构设计缺方法论（智能体数量凭经验）；**闭源 API / 未固定版本模型服务削弱可复现性**；评分聚合方法敏感（聚合方式不同可差 2 个严重度等级）。

## 四、怎么结合：7 个待验证的集成假设

> 以下多为"组件分别有人做过、但联合系统尚未验证"的设计推演（路线图），**不是会议已证实的闭环**；每条标注证据级别。

1. **智能体编排 × 实时知识追踪 / 多模态状态** — 把状态建模（#13/#56）与多模态信号（#23/#35）作为输入，让智能体决定"何时介入、给什么"。〔分组件已验证；联合闭环缺消融与真实课堂验证〕
2. **RAG + 本地量化 + 离线 = 降数据外流与成本的部署底座** — #84/#90/#55 分别验证可行。**本地部署 ≠ 零数据外流 ≠ 合规**；合规取决于日志策略、设备管理、身份权限、知识库治理与组织审计。〔分组件已验证；合规是制度问题〕
3. **可信评估闭环** = 多 LLM 陪审团（#52）+ 置信度校准（#11）+ 迁移评估（#22）+ 人在回路；回应"相同 F1 不同反馈"（#94）批判。〔各侧面单点有证据；整套闭环未联合验证〕
4. **确定性治理 + 智能体自主** — 治理边界（#42）+ 安全协调器（#98）；是把多智能体推向高风险场景的**必要但远非充分**条件（还需领域认证、合规、责任界定、人工兜底）。
5. **对抗过度依赖的设计范式合集** = 生产性摩擦（#22）+ 苏格拉底式（#9，增益不稳定）+ 脚手架渐隐 + 介入分级（#50）。〔注意：对抗式设计对低能力学生可能有害，是异质效应〕
6. **经典理论即架构约束** — 把理论写进损失/流程；〔有前景的设计范式，但不自动提升解释性或效果，也可能把错误假设编码进去，需消融与真实学习结果验证〕
7. **合成学生沙盒（快迭代）+ 真人多模态验证（慢校验）两段式** — 沙盒（#59）规避真实课堂伦理/风险，真人多模态对冲合成数据的乐观偏差。〔设计推演〕

## 五、提前规避的问题（落地清单）

**评估** — 别用单一 F1/AUC 下结论（要链到学习结果与构念效度，#94）；防时间/前瞻泄漏（#47）；加迁移评估（#22）；试点结果别当定论（本届过半为小样本）。

**LLM 落地** — 高风险场景必须人在回路（#30 的自动幻觉检测在其设置下 precision 仅约 73%）；数值/算数外包给计算器（#49）；提示复杂度匹配模型规模（#33）；复杂/高风险任务应拆分职责、加入可验证的中间结果与人工复核（多智能体只是可能的实现方式，非充分条件）；小模型加护栏防注入。

**自主性 / 交互** — 避免持续、无条件在线介入（保护生成性认知负荷）；对抗/辩论式设计是能力×任务的异质效应，对低能力学生可能有害（#22）；对话式 AI 带来"不知道该问什么"的新负荷（#19/#38）。

**可信 / 公平 / 部署** — 不应把"最大化信任"当目标（应追求校准）；拟人化在特定条件下与偏见相关；去偏见有副作用（#59 同一篇沙盒工作中，去偏智能体会牺牲反馈的同理心语气）；量化效果依赖硬件/框架/任务，不宜外推；闭源/未固定版本模型削弱可复现性；特殊需求/视障/听障/低资源地区学生不应在设计上默认排除（#1/#16/#82）。

## 六、容易被忽略的"生态 / 机构落地"视角

大会主题是 *Agentic Learning **Ecosystems***，但技术报告多停在"组件"层。真正决定能否进校园的，往往是下面这些本届着墨不足的维度：

- **"智能体" 不只是软件智能体**，还包括**学习者与教师的主体性**（可争辩反馈 #69、反思性参与 #80、求助行为 #101 勾勒了"学生如何理解、质疑、调节、退出 AI 支持"这条线）。
- **教师不是"只加材料"**：还要材料维护、反馈审核、课堂编排、误判兜底、专业发展与信任/负担管理。
- **机构集成才是真正的"生态"**：学习管理系统/学生信息系统集成、LTI/xAPI/Caliper、身份与权限、审计日志、与学校现有 IT 栈兼容——本届几乎未触及。
- **可信不止技术可控**：还涉及学生数据同意/撤回、监控边界、责任归属、学校采购问责。欧盟语境下，相关做法可能分别触及 **EU AI Act 的禁止或高风险条款**与 **GDPR 对未成年人、画像及可能的特殊类别数据的更高要求**（二者不是同一范畴）：用于评价学习结果、影响学习过程或考试违规监测的教育 AI 通常落入高风险；教育场景中的情绪推断某些用途可能被直接限制乃至禁止（安全/医疗目的除外）；自动评分与学生画像是否落入高风险，取决于用途、部署主体与后果。
- **可复现性是基础性问题**：模型版本漂移 + 闭源 API，对教育评估研究尤为关键。
- **成本要分层**：会上多报"单次推理成本"；学校真正关心的是总拥有成本（硬件、运维、培训、治理、升级、故障处理）。
