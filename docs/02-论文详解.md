### #1 Bridging the Silent Divide: AI Tutoring Meta-Analysis for Deaf & Hard-of-Hearing Learners（Amiruzzaman, Amiruzzaman, Dewan）— T1

- **研究问题**：〔领域问题〕AI tutoring 在大量"效果显著"的宣称下缺乏严格的循证检验，尤其对聋人/听障（Deaf & Hard-of-Hearing）及手语/语言学习这类边缘群体，AI 辅导是否真正有效、能否放大可及性（accessibility）与公平（equity）尚无系统证据。〔技术问题〕如何用 systematic review / meta-analysis 把分散研究的效应量（effect size）汇总、识别其可靠性与影响因素。**核心思路**：以系统综述 + 元分析量化 AI tutoring 的学习成效效应量，并特别拆出语言学习 / 手语学生的子群体对比，把 AI 定位为放大可及性与公平的"伙伴"而非替代。
- **主要方法**：**框架**：systematic review / meta-analysis，报告效应量大小（small/medium/large）及其与样本量的关系（小样本→不可靠）。**关键细节**：纳入约 18 项研究、约 245 个 effect sizes，数据库样本约 1,245 名 participants（以上数值因该 talk 录播音频严重残缺，转录可靠性较低，建议以 proceedings 为准）；研究目标为三项——检验是否真有效、检验其自称效果、理解影响因素；具体统计模型名与主测量工具名在录像中未能可靠还原（报告未明确）。
- **核心贡献**：① 针对聋人/听障与手语学习者这一被忽视群体，提供 AI tutoring 有效性的循证综述；② 给出子群体效应量对比的方向性发现——**手语 / 语言学习学生的 effect size 高于英语学生**（原因作者未解释）；③ 指出可用于该领域检测的 AI 应用类型尚少、领域不成熟，圈定大量研究空白。
- **研究启示**：**future work**：作者明确该领域成熟度低、研究空间大，需要更多覆盖手语及非英语语言的可及性导向研究，并解释子群体效应量差异的成因。**产品/落地启示**：教育 AI / ITS 产品在面向特殊群体（听障、手语使用者）时应以"放大可及性"为设计目标并保留人机协作（human-AI partnership）；上线前应有基于效应量的循证验证，而非仅凭"效果好"的宣称推广。

### #21 Modeling Adaptive Learner Trajectories in Immersive Environments via Interpretable Cellular Automata（Constantin, Bălcău）— T5

- **研究问题**：〔领域问题〕沉浸式学习的闭环自适应脚手架存在因果污染——学生 trace 既被用来选择支持动作，支持动作又反过来改变下一步 trace（t 时刻 trace 影响干预，干预影响 t+1 trace），仅看最终结果（outcome-only evaluation）无法解释 degradation / stabilization / recovery 如何随时间展开。〔技术问题〕如何让每个自适应决策 transparent / reproducible / accountable，做到步级可审计。**核心思路**：用紧凑、可解释的确定性元胞自动机（Elementary Cellular Automata）表示学习轨迹并驱动脚手架切换，使每步干预逻辑可追溯。
- **主要方法**：**框架**：把学习轨迹表示为 24-bit ECA 状态向量 x(t)∈{0,1}^24，每 bit 有预定义教学语义，分 Exploration（x0–x7）/ Stability（x8–x15）/ Risk（x16–x23）三簇；状态更新 x(t+1)=F_110(x(t))⊕G(i(t))，F_110 为 ECA Rule 110 的确定性局部演化，G(i(t)) 为由可观测 trace 经 XOR 计算、只允许翻转选定 risk bit 的修正向量；修正需 two-step persistence 去噪。**关键细节**：可观测 trace 含 task performance / latency / dwell time / navigation depth / engagement proxy / 应用特定风险信号；轨迹质量分 P(t) 由 performance + (1−归一化延迟) + engagement 加权；4 个确定性 scaffolding regime L0 minimal / L1 supportive / L2 reflective gate / L3 corrective，由 risk load 与 stability load 确定性映射选择。实验：4 个沉浸式应用（证据排序 / 跨情境社区图导航 / 治理场景规则学习 / 3D 媒体误解纠正），n=120、4 周、随机分 3 组各 40（静态基线 / 固定 regime / 闭环 ECA-110）。
- **核心贡献**：① 提出 24-bit ECA + Rule 110 + XOR 修正的**可解释闭环轨迹模型**，把自适应决策做到步级可审计（raw trace→activated bits→Rule 110 transition→XOR correction→regime）；② 实证显示闭环组在全部 4 个应用上一致占优（recovery time 更短、risk drift 更小、correction uptake 更高），且鲁棒性测试中 Rule 110 强于其他对照规则；③ 机制消融表明收益来自 supportive/reflective/corrective 三种干预的**交互**而非单一干预。
- **研究启示**：**future work**：作者自述需更大规模、更长、更真实课堂的部署；24-bit 状态刻意极简，存在表达力上限，未来需与现有 ITS 平台、多模态 trace 及更丰富学习表示集成，同时保持透明可复现。**产品/落地启示**：当自适应策略需要被审计或问责时，确定性、可解释的轨迹编码是对抗"黑箱自适应"的可行路线；但报告暴露关键落地缺口——如何把 CA 编码的状态翻译成教师可读的人类语言指标，Q&A 因网络问题未能正面回答（报告未明确），这是面向教师侧产品化的关键待解问题。

### #8 Early Indicators of Literacy Success: A Longitudinal ML Approach（Shamir, Pocklington, Tom, Houchins, Yoder）— T4

- **研究问题**：〔领域问题〕识字赤字若不及时干预会"滚雪球"（decoding→phonological awareness→comprehension/vocabulary 连锁恶化），但 at-risk 预测研究极少覆盖小学/初等阶段（primary level）。〔技术问题〕能否用 ML 从早期表现 + 学习时间指标，预测计算机辅助早期识字项目结束后 1 年和 2 年的学生成效。**核心思路**：用平台行为日志构造学习时间类与表现类特征，训练回归 + 分类模型做纵向预测，以 recall 优先帮助教师在影响最大的时机提前识别 at-risk 学生。
- **主要方法**：**框架**：基于 Waterford Early Learning（"Well"）自适应平台（pre-K 至二年级，本研究仅用阅读活动）的全量交互记录，建 end-of-year 分数回归 + 是否低于 benchmark 的分类模型。**关键细节**：样本约 5,500 用户（≥80% 相关数据），两 cohort——Cohort 1 = 2,772 名 K/一年级（DRA 测评 + 1 年后再测），Cohort 2 = 2,714 名 K 学生（学区测评 + 2 年后 NWEA MAP）；预测变量含 total/average weekly learning time、objectives encountered/mastered、overall 及 subskill（phonological awareness/phonics/comprehension/vocabulary/language concepts）平均分、grade level；训练/测试 70/30，5-fold CV；模型含 ensemble / 回归树 / logistic regression / gradient boosting / XGBoost / ridge / linear；指标 accuracy/precision/**recall（最重要）**/F1/ROC-AUC；建模过程借助 ChatGPT 辅助校准回归与生成 Python 脚本。
- **核心贡献**：① 在研究稀缺的小学早期识字场景给出可用的纵向预测工具（1 年分类 accuracy 84–87%、ROC-AUC 85–90%；2 年 accuracy 71–73%），以 recall 优先服务于有限教师资源下的早期识别；② 系统对比多模型——Ensemble 在 1 年分类综合最优、XGBoost/GBM 在回归最优，给出可操作的模型选型证据；③ 量化并讨论预测的时间衰减（1→2 年 correlation 与 R² 下降），暴露早期预测变量随时间失效的现实约束。
- **研究启示**：**future work**：作者自述需更精细的单区采样或扩大学区数，以分离时间差异与区域差异混淆的方差；外部建议指出预测窗口应尽量缩短（dropout 预测中 6 个月已太晚、3 个月数据即可留出干预时间）。**产品/落地启示**：教育 AI 产品做 at-risk 预警时，recall 优先于 accuracy 更契合"宁可多筛"的干预逻辑；但需正视长周期数据获取难（非营利机构难维持多年学校关系），以及把短窗口预测嵌入教师工作流以争取干预时间。

### #2 Designing Agentic Intelligent Tutoring Ecosystems for Primary Literacy（Hadjiathanasiou Diakou, Angeli）— T1

- **研究问题**：〔领域问题〕小学读写课堂的阅读理解是协作的、社会建构的、高度依赖教师指导，难以装进单一 tutor 系统。〔技术问题〕若不再把 tutoring 视为单一 tutor 交付，而是重新概念化为分布在生态系统中的 human-AI orchestration，能否提升小学生阅读理解、孩子认为哪些 tutoring functions 最有帮助、教师如何编排整个环境。**核心思路**：把中心化 tutor 拆为分布式 tutoring functions，由 teacher（编排）+ AI（反馈/生成）+ learners（创作）+ peers（互教）协同，用生成式 AI 支撑多模态创作与反馈。
- **主要方法**：**框架**：四类分布式 tutoring functions——cognitive scaffolding / feedback & regulation / representational mediation / orchestration fading；五阶段读写序列 engage→explore/analyze→create→communicate→reflect。**关键细节**：AI 两大角色——(1) 在开放/封闭题给反馈、答错给改进建议、允许二次作答提分；(2) 把学生续写叙事生成漫画视觉（AI 出图，学生填对话气泡）；多媒体创作用 Canva（获批用于小学，可加对白/音频/音乐/动效）。对象为真实小学读写课堂（希腊语教材）25 名四年级学生（9–10 岁），5 周干预；mixed-method 评估——前后测阅读理解（paired t-test）+ 参与度/动机问卷（Likert 1–5）+ 学生作品 rubric + 开放式回答 + 2 名教师课堂观察。
- **核心贡献**：① 提出把 ITS 重新概念化为**分布式 human-AI 编排生态**的功能设计视角，而非追求愈发自主的单一 tutor；② 实证显示阅读理解统计显著提升、大效应量，且困难学生提升尤为明显；③ 给出学生价值偏好的实证排序（最看重写自己的故事、把故事变漫画、加声效、用图片、为作品自豪——"feel like real designers or authors"），主题分析归为 creative agency / cognitive regulation / engagement。
- **研究启示**：**future work**：作者自述样本小、无对照组、时长短，计划做更大规模含对照组与跨校实验作为博士论文。**产品/落地启示**：面向低龄学习者的 ITS，价值可能更多来自"编排人类与 AI 多方能动性"而非更强的自主 tutor；落地的真实摩擦在报告中明确——需先向孩子解释"什么是 AI tutor"并安抚情绪、部分孩子缺乏计算机/打字技能需大量课外准备；实验设计上外部建议采用 two-round crossover/swap 以规避只给部分学生有益系统的伦理问题（第二轮须换不同文本以保统计可比）。

### #7 Decoding the Future: Predictive Rhyme Scoring for Literacy（Shamir, Pocklington, Tom, Yoder）— T4

- **研究问题**：〔领域问题〕早期儿童教育的 ML 预测严重未充分研究（某元分析中 <20% 文章涉及 K-12，另一分析近 10 年仅找到 1 项小学相关研究），而越早识别 at-risk 学生、干预越有效。〔技术问题〕早期押韵（rhyming）表现能否可靠预测后续语音意识（phonological awareness）成效，并用学年过半（mid-year）数据预测年末表现。**核心思路**：以 rhyming 这一已知的语音意识早期预测因子为输入，在学年中段做预测，为及早干预留出时间窗。
- **主要方法**：**框架**：基于 Waterford Well 平台的押韵活动数据，建分类（年末 phonological awareness objectives 是否 <80 分）+ 回归（预测年末 awareness 分数）模型。**关键细节**：初始约 15,500 名 pre-K 学生（2023–24 学年在家使用），分析样本收窄至约 6,650 名（≥80% 相关数据）；输入为 12 月 11 个 rhyming learning objectives 表现 + 特征工程衍生的 9 个变量（各 objective 的 mean/min/max/IQR/std 及早晚 objective 表现差异）；5-fold CV，训练/测试 70/30；模型 Random Forest / Gradient Boosting / LightGBM / XGBoost；指标 accuracy/precision/**recall（重点）**/F1/ROC；建模借助 ChatGPT 生成 Python 脚本。
- **核心贡献**：① 在公认空白的早期儿童 ML 预测领域，证明 mid-year rhyming 表现可预测年末语音意识（分类 recall 最高 85.7–88.7%，回归 correlation 约 0.76、解释约 58% 方差）；② 给出以 recall 为决策准则的明确模型推荐——尽管 XGBoost 综合指标最好，仍**推荐 Random Forest（recall 达 89%）**，即"可识别约 89% 的挣扎学生"并在学年过半提供 remediation；③ 再次实证 rhyming 作为 phonological awareness 关键成分的预测价值。
- **研究启示**：**future work**：作者自述未来应纳入 phonics knowledge 作为竞争预测因子对比，并扩展超出单一项目/单学年。**产品/落地启示**：以 recall 为首要指标的预警逻辑适合"早筛早干预"产品；报告暴露两个共性短板——特殊需求/阅读障碍学生（押韵困难者）未纳入数据集（应当考虑），以及 fluency 测量是行业级难题（需昂贵 audio data，且测 fluency 的模型无法同时测 comprehension）——外部评议指出若能从 rhyming 等变量预测 fluency 与 comprehension 将"带来很高的潜在社会价值"，是高价值但极难的方向。

### #55 Microclimate-Aware Educational Chatbot for Place-Based Climate Education（Khan, Blanchard, George）— T7

- **研究问题**：〔领域问题〕资源稀缺地区（教育员稀少的本地自然公园，目标推广到非洲、印度等专家/资源稀缺区）的气候教育如何规模化，且"AI 与气候变化并非最佳搭档"——需避免高耗能云端方案。〔技术问题〕小语言模型（SLM）能否在特定教育语境下与大模型（LLM）表现相当；教育者如何看待这种学习技术的有效性、可用性与教育价值。**核心思路**：以 place-based learning + microclimate 概念为切入点，用传感器实时环境数据情境化 RAG 生成，并以 frugal computing 部署小模型实现离线、低能耗。
- **主要方法**：**框架**：client→retriever（RAG 风格）→generator（LLM）的检索增强问答；核心是在检索环节注入一组传感器（温度/湿度/摄像头）捕获的实时环境数据→注入 prompt→生成情境化内容。**关键细节**：frugal computing 部署于 NVIDIA Jetson / Nano 级掌上设备，可跑 1–3B 参数模型、支持 4–5 人同时、太阳能供电、无需云/数据中心/联网；RQ1 用 2 篇环境科学论文、专家人工生成 10 组 Q&A，对比 Mistral 3B vs Mistral 大模型，用 BERTScore + Sentence-BERT 评语义相似度；RQ2 由 6 位教育者试用 + 问卷。
- **核心贡献**：① 提出并验证"微气候 + 实时传感器情境化"的 place-based 气候教育 chatbot，把环境实测数据接入生成以制造学习"紧迫感"；② 实证 SLM 与 LLM 在 context-specific 任务上表现接近、差异可忽略，为无需大模型/数据中心提供证据；③ 以 frugal computing（离线、太阳能、掌上设备）落地绿色 AI 路线，契合资源稀缺地区。
- **研究启示**：**future work**：作者自述缺 learner evaluation（进行中），并需改进纯文本 chatbot 的 UX/UI。**产品/落地启示**：在低资源、需离线自治的教育场景，"小模型 + RAG + 本地传感器"是可行的低成本部署范式，且教育者强调应作为人类中介的辅助而非替代（human-in-the-loop）；报告中明确的取舍——为保持离线自治刻意不做持久学生建模，个性化仅靠用户自报年龄/先验知识，反映边缘部署下个性化与隐私/自治的权衡。

### #34 Agentic Data Curation for Long-Tail Compiler Error Classification（Park, Kim）— T4

- **研究问题**：〔领域问题〕编程教育中稀有但与学习者相关的错误类型（long-tail error types，如结构性缺陷）若不能被诊断就得不到个性化反馈。〔技术问题〕compiler-verified augmentation 能否在极端长尾类上提升诊断覆盖率而不损失头部类准确率（coverage-accuracy tradeoff）；compiler 验证对 LLM 生成的训练数据是否必要；closed-loop 框架的收敛准则是什么。**核心思路**：用 compiler diagnostic 作 deterministic oracle 验证并闭环生成训练数据，使分类器覆盖稀有错误类。
- **主要方法**：**框架**：多阶段验证管线（MSVP）以 compiler diagnostic 做三重检查（代码是否真编译失败 / 编译诊断内容 / 诊断是否匹配目标错误类 taxonomy），并扩展为 class-agnostic 的自适应闭环数据创建框架（ACV）。**关键细节**：数据集 CS-EDM（某大学真实数据）+ CodeNet（IBM 竞赛编程），均长尾不均衡；评估 4 种损失函数（standard CE / class-balanced / focal / LDAM）、5 个随机种子，backbone 为 UniXcoder base + AdamW；生成器对比 Gemini 2.0 vs Gemini 3.0。
- **核心贡献**：① 提出 compiler 作 deterministic oracle 的闭环数据 curation 框架，最稀有类提升约 +16.9pp（远高于平均 +3.7pp）；② 给出"**verification rather than the scale of generation is the component that matters**"的核心论断——从 Gemini 2.0→3.0 消除 round-0 拒绝，表明拒绝反映类别难度而非生成器能力；③ 给出收敛/弃权准则（数据稀缺时行动、充足时 abstain 不增强），并主张错误分类应以 diagnostic coverage 而非平均准确率评估。
- **研究启示**：**future work**：作者自述仅适用编译期错误（运行时错误超范围）、仅覆盖 Java/C 迁移性存疑、用离线数据集未在真实课堂验证。**产品/落地启示**：在有确定性 oracle（如编译器、单元测试）的领域，"验证优先于生成规模"是构建可靠训练数据的高性价比原则；Q&A 暴露一个一个可行的简化路径——既然已有确定性分类器，何不直接用于学生代码反馈而非仅做数据增强（作者认可可行，列为 future work），提示该确定性组件本身即有直接落地价值。

### #40 Conditioning Context Shapes Stateful Personalization in Educational Recommender Systems（Park, Medhat, Wai, Thajchayapong, Goel）— T1

- **研究问题**：〔领域问题〕教师面向的教学推荐系统中，个性化常被等同于"对当前问题的响应"，但当持久学习者状态（persistent learner state）成为 conditioning context 的一部分时，推荐行为如何变化尚不清晰。〔技术问题〕如何在固定 student question、model、recommendation process 的前提下，隔离单一变量（conditioning context）来观测个性化行为的改变。**核心思路**：用受控对比把"仅当前问题"（context-based）与"加持久学习者状态"（memory-based）两种 conditioning context 隔离开，证明仅改输入信息即可让固定模型产生不同个性化行为。
- **主要方法**：**框架**：stateful personalization 三组件——learner representation→instructional tactic selection（基于 Hattie's feedback 学习理论）→recommendation generation。**关键细节**：数据来自 Jill Watson（Georgia Tech 真实课程部署的对话式 AI 助教）的师生交互日志，隶属 Georgia Tech "AI Architecture for Learning"项目；提出量化诊断指标 deviation correlation（学生问题偏离课堂常态 vs 推荐偏离推荐常态的相关性）；教师面向 dashboard 由 5 位教师做 co-design + UMUX-lite 可用性反馈；learner representation 的构造/验证在 adjacent work 完成（本研究未含）。
- **核心贡献**：① 用受控实验隔离 conditioning context 单一变量，证明"learner representations shape conditioning context; conditioning context shapes recommendation behavior"——同一问题对两个不同学习者可分别得到 engagement-oriented 与 performance-oriented 推荐；② 提出 deviation correlation 作为可量化诊断指标，显示引入 learner state 后行为从 question-driven 转向 learner-driven（耦合度下降）；③ 提出"lower correlation ≠ worse personalization、responsiveness ≠ personalization"的概念辨析。
- **研究启示**：**future work**：作者自述未实地部署收集学生 outcome，仅有 dashboard 的教师可用性反馈。**产品/落地启示**：对 LLM-based 教育推荐系统而言，仅改变注入模型的学习者状态即可改变个性化行为，是低成本的个性化杠杆；但报告明确暴露定位风险——多位听众反复追问其与 Knowledge Tracing（含 Deep KT）的差异，作者未充分交代与知识追踪文献的关系、论断（"现有系统只用当前问题"）可能不准，提示该方向产品化前需厘清与经典学习者建模的关系与增量价值。

### #86 A Specialized Conversational Agent for Peer Assessment Training (Propositional Logic)（Badea, Popescu, Temperini）— T1

### #44 Argument Quality in the Eye of the Annotator: Understanding Annotation via Eye Tracking（Ivanova, Bektas）— T4

- **研究问题**：〔领域问题〕为"给论证文写作提供自动质量反馈"的工具创建高质量 gold-standard 数据集时，标注者难以达成一致，其认知复杂性与认知负荷来源不明。〔技术问题〕标注者读 annotation guidelines 时注意力是否显著下降；短 vs 长 guidelines 哪个被感知为认知负荷更高；整个标注流程中注意力在哪个点显著下降。**核心思路**：首次用眼动追踪（fixation + saccades）贯穿标注全流程，量化注意力随指南与任务进程的变化，据此指导标注界面设计。
- **主要方法**：**框架**：randomized crossover design（一半先看 minimal、一半先看 extended guideline），两轮迭代，每轮 4 步——读指南→标注 10 段文本→填 NASA-TLX（测心理负荷）→领域问题。**关键细节**：眼动两类数据 fixation（注视点/注意顺序）+ saccades（眼跳，反映分心/疲劳/阅读流）；标注尺度对比 absolute(Likert) vs relative(成对比较)，标注主体采用更现实的 crowdsourcing（对比稀缺昂贵的 experts）；40 名参与者（教育背景多样、英语熟练），数据来自该领域常用数据集。
- **核心贡献**：① 首次用眼动揭示计算论证标注的认知复杂性，证明注意力随读指南显著下降、但插入示例/中断时显著回升；② 反直觉发现——长指南被感知负荷更高（NASA-TLX 唯一显著差异），但读完后标注更自信，说明"detailed-but-demanding"指南有正面价值；③ 发现注意力在任务过半前（约 25–30 分钟）就显著下降，比预期早，建议把标注切成小块、限制单次时长。
- **研究启示**：**future work**：作者建议未来加入更多 interruptions / 交互式标注以维持注意力。**产品/落地启示**：对依赖人工标注/众包构建训练数据的教育 AI 流程，标注界面应分块化、限时、穿插示例与中断以对抗认知疲劳；详细但更费力的指南可换来更高标注自信，值得在数据质量与标注成本间权衡。报告注明眼动数据需排除 16/40 名参与者（隐形眼镜/厚镜片致眼位偏移），手动校正后无显著差异但仍剔除、保留其问卷数据。

### #50 Helping or Not Helping: A System to Increase Students' Critical Thinking. A Pilot Study（Matarazzo, Sciarrone, Martini, Caldelli）— T1

- **研究问题**：〔领域问题〕在 LLM 驱动的生成式 ITS 中，"永远在线（always-on）的即时辅助"是否会无意中减少学生自主推理与批判性思维的机会——即 "If AI can always help, should it always do so?"，作者将其提为 "assistance dilemma（援助困境）"：更多帮助提升短期任务表现（更快、更少挫败），但过度帮助降低学习者自主性。〔技术问题〕如何在 tutor 架构中把"何时介入、何时沉默"显式化为可调节的介入策略，并用交互动态（而非仅 performance）评估其效果。**核心思路**：贡献不是更好的语言模型，而是决定语言模型何时介入、何时保持沉默的机制——"Non-intervention is not inactivity; it is a pedagogical design decision."
- **主要方法**：**框架**：Intervention Governance Framework（介入治理框架），四层 tutor 架构——① Interaction layer（管理答题/求助/换题等事件）；② Learner state layer（维护尝试次数、连续错误、求助请求、用时 + 主题/难度元数据）；③ Intervention-level governance（核心，实现 escalation rules、non-intervention constraints、error-based triggers）；④ Generative engine（按介入级别 + 题目上下文生成内容）。三级介入输出：H0=no intervention、H1=minimal conceptual orientation（轻量概念引导）、H2=explicit procedural guidance（明确步骤）。**关键细节**：within-subject 设计，30 名学生（12–13 岁），条件顺序随机化，两条件对比——Baseline（帮助即时可得、escalation 仅基于请求、强介入无限制、on-demand）vs Self-regulating（介入受调节、帮助可延迟、escalation 取决于学习者行为、强介入受约束）；30 道算术题（15 easy + 15 hard），覆盖 operator precedence/parenthesis/fractions 三主题，每生每题只做一次（两条件间平衡拆分）；行为指标为 number of attempts before solution、help requests、error persistence、time on problem（不只看正确率）。
- **核心贡献**：① 提出把"介入时机/介入强度"形式化为可治理策略的 Intervention Governance Framework（H0/H1/H2 + escalation/non-intervention 约束），将研究焦点从"生成什么"转向"何时生成"；② 实证显示整体任务表现在两条件间相对稳定、但交互动态改变——Self-regulating 下更少即时求助、更大坚持性、更多自主尝试，并出现反直觉现象：强介入(H2)略增（学生坚持更久才求助，介入更少但一旦发生更易升级）；③ 给出难度调节效应——hard 题目效应更强，提出"困难任务中 premature assistance 更易替代而非支持自主推理"的理论含义，并主张生成式 ITS 应同时用 interaction dynamics 而非仅 performance outcomes 评估。
- **研究启示**：**future work**：作者承认当前只测即时行为、未测 durable learning，next step 为 cohort 对比 + 无 AI 的 take-home post-test 测真实学习提升/迁移；介入策略对所有学生相同，下一步可按学习者画像/自主水平个性化介入策略本身。**产品/落地启示**：对教育 AI 产品，"何时不帮"应被当作显式的产品策略而非缺省即时响应；可借鉴"延迟介入 + 基于 learner state（错误/尝试/求助）触发升级"的治理层设计，尤其在高认知负荷任务上更需克制。报告中明确的方法学边界：样本小（pilot，30 人）、领域受限于算术、未直接测量批判性思维本身（只用行为代理指标）、未测 learning gains/transfer——产品决策不应据此 pilot 外推长期效果。

### #30 Multi-Agent Intelligent Tutoring with 4D Personalization and Hallucination Detection（Inoubli, Boussema, Cazenave, Zeghal）— T1


- **研究问题**：〔领域问题〕引 Bloom(1984) 1对1 辅导可把学生从平均提到 top 2%，但现有 ITS 四大问题——只适配 1–2 个学生特征（忽略情感/行为）、只在 topic 层追踪学习（漏具体技能缺口）、LLM 生成内容验证不足、单一 monolithic LLM 易幻觉/信息过载。〔技术问题〕(1) 细粒度 multi-agent 架构能否优于 3–5 agent 系统？(2) 多维（4D）个性化能否提升学习？(3) 混合幻觉检测能否在保证质量的同时把 latency 控制在 <2s？**核心思路**：用细粒度的分层多智能体协作替代单一 monolithic LLM，以 17-agent 架构承载 outcome 级掌握度追踪、4D 个性化与混合幻觉检测。
- **主要方法**：**框架**：17-agent / 5 层架构——L1 数据收集（建学生档案）→ L2 intelligence（算 per-learning-outcome/per-skill 掌握度，经 Gemini 生成推荐）→ L3 recommendation（选练习/调难度/排 7 天计划）→ L4 feedback（自适应反馈 + 重构解答）→ L5 evaluation（验证所有 LLM 生成内容、追踪指标）。**关键细节**：Orchestrator 提供 4 种工作流（sequential 5s / parallel <2s / conditional / priority-based）+ 4 种 agent 选择策略 + graceful degradation（97% 可用性）。三项创新：① learning-outcome 级掌握度公式（accuracy 基分 + volume bonus + recency factor，封顶 100）；② 4D 个性化——Cognitive（保持 60–80% 准确率区间，streak/rolling accuracy/flow-state 三机制，89% session 在最优区 vs 规则系统 62%）、Affective（不询问即检测 motivation/confidence/anxiety，映射 5 种情绪态，priority trigger 触发 Gemini 个性化消息）、Behavior（按 session 时长分 bin 推荐最优时长，检测疲劳→更短 session）、Temporal（按时段评分，在个人高峰窗学习者得分高 40%）；③ 混合幻觉检测——semantic entropy（对同一问题生成 5 个回答按语义分组测分歧，>0.8 标记）+ word-level 置信（CCP score，>30% 低置信标记），加权 60% entropy + 40% word-level、最终 >0.5 标记，该 60/40 配置 precision 最高 73%、假阳率最低 2.4%。评估：4 周研究，47 名学生（数学/精算/大数据+AI 三专业），内容 461 题/37 练习/4 learning outcomes/23 sections，全自愿不计分；选 SOA 金融数学（不在任何受试课程中）作内部效度论据。
- **核心贡献**：① 提出 17-agent 五层细粒度架构 + orchestrator（多工作流/选择策略/graceful degradation），并以"agent 失败则全停"等机制实现 97% 可用性、latency 降 28%；② 提出认知/情感/行为/时间的 4D 个性化，其中情感维度无需询问学生即从交互信号检测，并实测把 session 维持在最优准确率区的比例从 62%（规则）提到 89%；③ 提出 semantic entropy + word-level 置信的混合幻觉检测（60/40 加权），在约 800ms 内达 73% precision、2.5% 假阳率，把内容可信性纳入辅导闭环。结果：在 4 周试点中观察到前后测提升（作者对三项研究问题均给出肯定回答）；但该研究**无对照组，不能据此作因果归因**，具体数值以正式论文为准。
- **研究启示**：**future work**：作者自述外部效度有限（47 人/单领域/4 周）、无对照组、无正式 ablation；列 8 个方向——对照试验、3/6/12 月留存、跨学科（编程/科学/人文）、可解释 AI、可扩展性等；数学是优势域，能否泛化到 essay writing/语言生成待验证。**产品/落地启示**：多维度个性化（尤其无打扰的情感检测）+ 内容级幻觉检测 + graceful degradation，是面向高可信学科辅导的可借鉴组合；但报告暴露关键方法论缺口——agent 数量/层数靠经验式按 use-case 定（"有想法就加 agent"），无定量度量、跨领域可移植性未知，且对教师提供的教学材料有较强结构化要求（需带难度等级/平均用时/分步纠错说明），教师界面与"最少题量"标准仍是 future work，产品化前需补这些工程与方法论空白。

### #68 An Agentic AI Architecture for Nutritional Education（Orciuoli, Pascuzzo, Damiano）— T2

- **研究问题**：〔领域问题〕用 LLM 生成个性化膳食计划（meal plans）并教学时，存在 hallucination（营养值未验证、来源不明）、无临床互操作（健康状况定义歧义）、无教学结构（交互是 one-shot）三大默认风险。〔技术问题〕如何把 LLM 生成的膳食计划框定为一个 Intelligent Tutoring System，针对高血压/糖尿病等特定健康状况用户生成饮食并教学。**核心思路**：用三视图 agentic 架构把 LLM 嵌入更大系统——以 MCP 工具接入结构化营养/临床数据保证可靠性，以 ITS 四支柱赋予教学结构。
- **主要方法**：**框架**：三视图 agentic 架构——Conceptual view 基于 CoALA（memory/planning/perception/action/tools，memory 分 short-term 与 long-term）；Technological view 用 MCP（Model Context Protocol）+ FHIR，每工具映射专用 MCP server（本地 FHIR server 取 patient/condition/allergy resource、AutoCook server 生成 recipe、Open Nutrition server 取营养成分）；Pedagogical view（核心贡献）按 ITS 四支柱框定——user interface / tutoring model（LLM orchestrator + planning + procedural memory）/ domain model（专家知识=long-term memory，经 MCP 实现）/ student model（病史与交互史，更新 episodic memory），教学策略用 microlearning + knowledge tracing。**关键细节**：LLM orchestrator 驱动 cyclic decision making（可用 ReAct / Tree-of-Thoughts），工具流为 get user data→读 FHIR 条件定"避免项"→AutoCook 生成→算营养→判断是否达标（如 <2000 kcal）否则重循环；case study 用 Synthea 生成 10 名高血压患者（参考 DASH diet，sodium <1500 mg/day），模型用 GPT-OSS 20B，baseline 为无外部工具的同模型简单调用；评估基于 Kirkpatrick 四层（Reaction/Learning/Behavior/Outcome），其中 Learning 为重点，用 knowledge tracing（4 级 beginner/elementary/intermediate/proficient）经多选题评估。
- **核心贡献**：① 提出把营养教育系统框定为 ITS 的三视图 agentic 架构，用 ITS 四支柱给 LLM 膳食生成补上教学结构（Pedagogical view 为核心贡献）；② 用 MCP + FHIR 接入结构化临床/营养数据，使膳食营养值精确、健康状况无歧义识别，并保持 traceable path（不给模型完全自主访问）；③ case study 显示提出系统更多落入 sodium optimal range，而 simple LLM baseline 因"知道高血压要低钠"而过度纠正、产出不现实的过低钠膳食。
- **研究启示**：**future work**：作者自述 case study 基于合成画像（Synthea）；系统非完全自主 agent，需显式 user prompt 激活（reactive），完全 agentic 需 proactive behavior（条件更新时自动激活）；评估需补定量指标 + 纵向用户研究。**产品/落地启示**：把 LLM 框定在更大架构中、用专用工具接入权威结构化数据，是抑制幻觉、保证（营养/临床）可靠性的有效模式，仅优化 prompt/invocation 不够；报告中明确的合规缺口——当前 pilot 未加入伦理约束/医疗对齐要求，但认知层预留了引入此类约束的位置，提示健康类教育 AI 上线前须补医疗对齐与伦理护栏。

### #17 A Comparative Analysis of Agentic AI Frameworks for Personalized Academic Advisors（García-García, Matilla-Molina, Dodero, Muñoz）— T1

- **研究问题**：〔领域问题〕高等教育学术建议（用成绩/兴趣/要求/职业目标推荐课程与路径）需要 reliability——要 grounded/coherent 的推荐而非 fluent but unsupported。〔技术问题〕哪个 agentic AI 框架（而非单一工具）最适合构建可靠的个性化学术顾问。**核心思路**：用 role-based multi-agent debate + validation + consensus 替代单一 monolithic LLM 以减少幻觉/信息过载，并用两阶段评估在多个框架中选型。

### #69 Adaptive Research Student Mentoring via SRL-Aligned and Contestable AI Feedback（Adeyemo, Kikuchi, Ozono）— T1

- **研究问题**：〔领域问题〕研究生进度管理环境特殊（过程开放、里程碑不清、导师反馈稀疏，高度依赖自我监控与自调节），现有系统存在两个 gap——SRL coverage（多围绕结构化课程作业、偏重 performance 阶段，对 forethought 与 self-reflection 支持不足）与 contestation（把学生当作 AI 反馈的被动接受者，无正式机制挑战 AI 评估）。〔技术问题〕AI 能否为研究生周报生成覆盖 SRL 全三阶段、嵌入 metacognitive scaffolds 的反馈（RQ1）；系统架构能否支持 feedback contestation（处理申辩、产出 evidence-grounded 决策、基于已验证挑战调整行为）（RQ2）。**核心思路**：用双 agent 架构把 Zimmerman SRL 三阶段支架与可争议（contestable）反馈结合，使学生既被引导自调节、又能正式挑战 AI 评估并触发系统自适应。

- **核心贡献**：① 提出覆盖 SRL 全三阶段（含被忽视的 forethought 与 self-reflection）、嵌入 metacognitive scaffolds 的研究生 mentoring 反馈系统；② 提出 contestability 机制——把学生从被动接受者变为可正式挑战 AI 评估的主体，并通过 memory rule 实现长期对齐（escalation 后存规则、后续相似挑战直接拒绝不再升级，证明 adaptive behavior）；③ feasibility 实证 feedback quality 强（判官间 100% within-one-point agreement）、但 rubric alignment 较低，由此得出"**RSMS 作为 mentoring feedback 系统比作为 grading 系统更强**"——这一发现正是 contestability 设计的动机。
- **研究启示**：**future work**：作者自述报告为历史数据（predate 系统，致 "prior advice integration" 分数不可解释）、数据仅 2 名学生、挑战案例为 LLM 生成、adaptive refinement loop 仅 1 个 escalation case 测试；live deployment 计划纳入真人导师对比 LLM-judge 评估。**产品/落地启示**：对开放式、低结构的学习场景（研究生指导），SRL 全阶段支架 + 可争议反馈 + 人类升级（human escalation）是把 LLM 反馈做成可纠错/可解释/长期对齐机制的可行路线；报告中明确情感诉求或主观努力不是改分的充分依据（supervisor preference rule），且因 LLM 评分 rubric alignment 不稳，把系统定位为 mentoring 而非 grading 更稳妥；当前仅用 LLM-as-judge（虽用 supervisor persona 但无真人导师），落地前需真人评估校验。

### #49 BacPrep: Lessons from Deploying an LLM-Based Bacalaureat Assessment Platform（Dumitran, Dita, Dumitran）— T7

- **研究问题**：〔领域问题〕罗马尼亚高考 Bacalaureat 的自动评分如何在 high-stakes 场景可靠落地——社会动机是普遍私教造成的教育不平等（引一项 EduPedu 研究称 50% 高中生靠私教备考），希望 LLM 给无优质教育资源（尤其农村、贫困）的学生免费工具、做 "digital equalizer"。〔技术问题〕把整份考试一次性喂入单 prompt 的 monolithic prompting 为何不适合 high-stakes 评分，应如何重构。**核心思路**：这是一篇明确定位的 "failure presentation"——不修补使其勉强可用，而是推倒重构 + 收集更多数据集 + 造合成数据，并提炼可复用的失败教训。
- **主要方法**：**框架**：初版（"fast and dirty"）用 Gemini（<1M token 上下文）把整份考试（official subject + grading scheme + 学生答案）一次性单 prompt 评分、数据存 JSON；重新设计转向每学科/每题型专用 grader。**关键细节**：覆盖 math/computer science/Romanian 三学科（团队具备相应评分专业能力）；初版收集约 200 份真实学生答案（限定特定年份保证真实 + 多年数据量）后因质量不足停止。current 版本：specialized grader per subject/curriculum/problem-type，math 用专用 grader（可 fine-tune 或基于专家解答运行），同一题做 3 次多轮运行取中位数（median of 3）以稳定（曾出现同题 4 分/6 分不一致），数学计算交给计算器（不让 LLM 算数），并引入 OCR（学生愿拍照上传手写解答以提升 adoption/数据收集）；副产品是衍生出多篇论文与多个数据集（Bac 全学科/Romanian/biology/合成数据集）。
- **核心贡献**：① 系统化记录并论证"monolithic prompting 不适合 high-stakes 评分"——一份卷约 10–15 道异质题、结果长达数页、每题评分本身是独立任务，需拆为 specialized grader per 题型；② 暴露并对治 LLM 评分的具体失效模式——数学算数极差（把应为 9 分加成 17 分，结果时而翻倍/减半）、幻觉式给分（学生未作答时模型自己解题再给满分）、Romanian 评分只看内容呈现而忽略语法/标点/结构、官方评分标准颗粒度不够需自行细化（每点给 1 分），并以"算数交给计算器 + median-of-3 + 专用 grader"缓解；③ 以诚实的失败叙事提供方法论价值——"a talk about failure... helping you fail in different ways and not in this way"。
- **研究启示**：**future work**：计划扩展数据集训练评分模型（fine-tune 或 RAG + 专家解答）后重新部署，数据足够后由 expert system 创建验证。**产品/落地启示**：在 high-stakes 教育评分产品中，即使标注为"实验性"也不应给学生很差的结果（哪怕仅训练用途），故平台技术上免费在线但因质量不满意刻意未推广；可复用工程经验包括——LLM 不做算数（交给计算器）、同题多轮取中位数稳分、按题型拆专用 grader、用 OCR 降低录入门槛以提升采纳与数据回流。

### #59 The Pedagogical Sandbox: Pre-deployment Validation of an Agentic AI for Formative Feedback via Synthetic Student Interactions（Vittorini, Cofini, Letteri, Jobe）— T6


- **研究问题**：〔领域问题〕agentic AI 行为不可预测（举例：某 agentic 系统因无法完成工作而 误删整个数据库），直接放进真实课堂风险严重；且新方案部署前需小规模测试，但出于伦理需申请 伦理审查（耗时），快速迭代受阻。〔技术问题〕能否用 synthetic students（合成学生 agent）在完全模拟的虚拟课堂中、于真实部署前测试 agentic 方案并提前发现风险/异常。**核心思路**：搭建 Pedagogical Sandbox（教学沙盒）作为 proactive diagnostic——用从历史学习分析数据实例化的合成学生，在 design-based research 的 micro-cycle 中验证被测系统是否真在正确优化、并提前暴露问题。两个 RQ：RQ1 合成学生+沙盒能否真实模拟学生行为？RQ2 被测 agentic 系统是否真在正确优化、适配不同学生？
- **主要方法**：**框架**：multi-agent 基础设施用 JADE（Java agent 平台，支持 ACL/标准通信协议）+ LangChain + Ollama/Llama；合成学生从 learning analytics 实例化并被指令 "simulate"。**关键细节**：被评估系统（agentic feedback optimizer）含协作 agent——Profile agent（分析学生能力/tone、定义最佳沟通语气）、Solver agent（生成反馈+练习+解答）、Bias neutralization agent（识别并消除 LLM 偏见）、Prompt agent（修订 solver 的 prompt）。模型按 micro-cycle 选最佳候选：合成学生生成"合理错误"用 Qwen（最佳，DeepSeek/Mistral 也是候选）以模拟学生评价反馈"清楚/不清楚"；其余 agent/交互用 GPT-OSS 20B 配 prompt engineering（chain-of-thought 等），所有 prompt 公开。研究方法为 Design-Based Research + continuous improvement（micro-cycle）；模拟 formative assessment 闭环（做练习→给解答→收反馈→评价→循环），实例化不同合成学生执行、收集 179 个交互，由专家评估 plausibility（RQ1）与 prompt 质量（RQ2）。
- **核心贡献**：① 提出 Pedagogical Sandbox + synthetic students 的"部署前验证"范式，在正式课堂研究前用于低风险预检（不替代伦理审查），作为 agentic 教育方案的 proactive diagnostic；② 实证有效——RQ1 专家评估合成学生交互在 92.5% 案例 plausible（评估者间 robust agreement，且观察到 non-monotonic learning curves，像真实学生），RQ2 prompt 在 92% 案例按学生画像+required zone 正确优化；③ 提前数月暴露了在常规 micro-cycle 中未被发现的问题——关键发现是 Bias mitigation 的代价：tone agent 建议 warm/empathic、solver 反馈确实温暖，但 bias neutralization 完全改写反馈使其变冷（cold），移除偏见却牺牲同理心；并发现偶发 scaffolding mismatch（给低水平学生推荐过难材料）。

### #52 An Agentic Evaluation Architecture for Historical Bias Detection in Educational Textbooks（Ștefan, Dumitran）— T6

> 说明：部分内容（含 Q&A 与完整模型清单）未完整记录，以正式论文为准。

- **研究问题**：〔领域问题〕教科书是"truth baseline"、塑造身份与社会观，但审查工作量巨大——罗马尼亚有 1,641 本部批教科书（数十万页）、每 4–8 年须更换、错误不可避免（2026 年底有大课改）。〔技术问题〕如何用 multi-agent pipeline 高召回地筛出历史偏见/争议内容、又控制假阳性，并区分"引用历史文本"与"作者立场"。**核心思路**：目标不是把人移出审查流程，而是辅助、加速审查，让专家聚焦最重要问题；以分阶段 agentic 架构（高召回 screening + source attribution + 多 LLM jury + meta-juror 聚合 + human escalation）实现。
- **主要方法**：**框架**：Multi-agent pipeline，输入教科书 PDF（数字版或扫描版均可），输出非技术人员可读的最终报告，每本 <30 分钟、token 成本 <$2。**关键细节**：数据集为 16 本当前在用的历史教科书（报告的每个错误都正在影响学生）。架构分阶段——① Multimodal screening（高召回，multimodal LLM 把 PDF 当图像、每 5 页一块输入；用 multimodal 而非文本提取库是为保留 sidebars/captions/margin notes）；② Source attribution classifier（关键创新，把文本分类为 author's original text vs primary source material，确保"为教育目的引用 ≠ 认同所引内容"，解决 pilot 中历史人物引语被误判为作者争议叙述的问题）；③ Multi-agent jury（5 个不同 LLM 独立行动、各有不同 reasoning architecture，被 prompt 对可能假阳性给低分以补偿高召回偏差；用 7-point Likert——先试 10 点发现选项越少分类越好故降到 7 点，每 juror 输出 JSON 含 source attribution flag + taxonomy label + 7 点 severity + confidence score + explanation；taxonomy 为 4 大类共 15 个 fine-grain labels）；④ Meta-juror 聚合（基于 confidence 聚合，测两种方法——数学聚合按 confidence>0.7 过滤后算 weighted average severity、任意两 agent 差>1.5 则升级人工；独立审议则让 meta-agent 评估 5 个 reasoning 后自行推理输出，发现重大分歧时升级）。对比示例：4 个 juror 评 severity 2、1 个评 5，数学法因该 5 分 confidence 低被丢弃→加权得 3（minor），独立审议法发现 juror-5 高 confidence→输出 5（significant），两法相差 2 个等级，故方法选择极重要。
- **核心贡献**：① 提出端到端的 agentic 教科书偏见审查架构，高效（<30 min、<$2/本）且兼容扫描版与数字版，定位为加速人工审查而非取代；② 关键创新 Source attribution classifier——把"引用 vs 认同"显式分离，纠正高召回 screening 对历史引语的系统性误判；③ 揭示多 LLM 评分聚合的方法敏感性——confidence-weighted 数学聚合与"独立审议"meta-juror 可导致 2 个严重度等级（minor↔significant）差异，并以 confidence 加权 + 分歧阈值 + human escalation 作为治理设计。
- **研究启示**：**future work**：（视频截断，limitations/完整模型清单/Q&A 未录到，报告未明确）。**产品/落地启示**：对内容审核类教育 AI 产品，"高召回 screening + 后续低分 prompt 过滤 + 多模型陪审 + confidence 加权聚合 + 人工升级"是可借鉴的分层架构；两条经验值得复用——量表选项越少（7 点优于 10 点）人机一致性越高；多模型聚合方法的选择对结论影响巨大，需明确并固定聚合策略。本研究在 16 本在用教材中未发现 harmful 内容，更关注 pedagogical harm，提示此类工具的现实价值在于"减负"而非"判罪"。

### #83 From the Zone of Proximal Development to ITS: Towards an Adaptive Artificial Pedagogical Agency（Ba, Hotte, Henni, Diop）— T1

- **研究问题**：〔领域问题〕在生成式 AI 时代如何概念化"人工教学能动性（artificial pedagogical agency）"——经典 ITS 大多 scripted、聚焦内容而非学习过程、对社会与情感动态不敏感，实现的是被批判为 "frozen ZPD（冻结的 ZPD）" 的状态：用预设规则判定水平、给标准化帮助，无法建模"内化（internalization）"的时间维度。〔技术问题〕如何把 Vygotsky 的 ZPD（被作者定位为"诊断概念而非教学方法"）、Bruner 的 scaffolding+fading、situated mediation 落到一个可设计、可评估的技术架构上。**核心思路**：提出 AAPA 这一概念架构（明确"不是单一系统，而是设计目标与评估透镜"），用四组件闭环把经典学习理论与现代 AI（HMM/RL/memory）桥接。

### #32 Combining Fuzzy Graph Matching and KNN for Educational Assessment（Zitouni, Bezine）— T4

- **研究问题**：〔领域问题〕如何为年幼学习者（5–11 岁）提供自适应、实时的阿拉伯语手写纠错反馈。阿拉伯语难点：含大量 diacritic marks（变音/附加符号，形状相同仅变音符不同即彻底改变词义）、书写右到左且连笔（cursive），对儿童尤难，手写识别远难于印刷体（提问环节指出印刷体 OCR 已达 95–97%）。〔技术问题〕如何在手写固有的不精确性/不确定性下做稳健的字形分类，并把分类结果转化为有指导意义的纠错反馈（目标"不仅是分类，而是帮助儿童学会书写"）。**核心思路**：用 Fuzzy Attributed Relational Graph（FARG）表示手写笔段结构，结合 Fuzzy Graph Matching 与 KNN 做分类，再按多类错误加权生成反馈。
- **主要方法**：**框架**：data acquisition → preprocessing & segmentation → 图表示（FARG）→ 两任务：分类（Fuzzy Graph Matching + KNN）与反馈。**关键细节**：自研移动 App 捕获 online 手写轨迹，存为 XML（含 participant info + (X,Y) 坐标 + 第三维 pressure 0/1 即 pen up/down，按 pressure 切分笔段）。数据集为突尼斯一校 50 名参与者（5–11 岁，15 男/35 女，几乎全右利手，均无学习/书写障碍，无书写速度/大小约束）。图表示：每个 segment 为节点、segment 间关系为边；节点属性两类——size（基于 bounding box 的 equilibrium distance + 模糊逻辑分 5 档）与 type（simple/complex，基于 Beta-Elliptic model 把每段表征为特征向量、按 deviation angle θ 在三角圆上编码 perceptual code，再用 genetic algorithm 得 global perceptual coding，共 10 个 basic chip）；边属性仅 1 个 spatial relationship（从两段间 6 个几何特征经 fuzzy controller 得 10 种关系，分 directional 与 topological）。分类：按 segment number 分组训练图，算 fuzzy graph similarity = node + edge similarity（用搜索算法只取 optimal path 降复杂度），找最近邻并算 fuzzy membership degree、arg max 赋类，最优 k=5，引入 epsilon 防除零。反馈：检测 5 类错误（direction/order/stroke number/shape division/spatial relationship），加权得 global feedback（值域 0–1），映射 3 档 legibility（poor <0.6 / medium 0.6–0.8 / high >0.8，区间由专家定义）。
- **核心贡献**：① 用 Fuzzy Attributed Relational Graph 把手写字形建模为带模糊属性的图，以模糊逻辑系统化处理手写的不精确性/不确定性，是面向非拉丁、连笔文字的领域特化方案；② Fuzzy Graph Matching 与 KNN 联合分类，实测分类准确率 89%、错误类型检测 89.7%、legibility 与专家一致性 92.1%，并优于 KNN-only / weighted distance 基线；③ 反馈导向设计——不止给类别，而是定位 5 类可解释错误并加权（最高频为 direction error 34%），附带实验显示 visual+animated feedback 能帮助儿童识别并纠正高频错误。
- **研究启示**：**future work**：作者计划用 deep learning 提升错误检测精度、加 reading skills 模型、从字符/字母扩到 phrase、扩充数据库 lexicon，并探索 fuzzy graph embedding（把每个 segment 自身也表示为图）、迁移到数学表达式识别；Q&A 中收到从 fuzzy type-1 升级到 type-2 以提精度的建议（未实现）。**产品/落地启示**：面向低龄、非拉丁文字书写训练的 ITS，通用识别方案不足，需领域特化（模糊图 + 专家定义的 legibility 区间 + 辅助基线），且产品价值在可解释的逐类纠错反馈而非单纯打分；报告亦暴露其局限——系统一次只识别单词/字母（基于逐类相似度），且对 threshold/epsilon/边界的参数敏感性缺乏系统分析（Q&A 未给出敏感性数据），是产品化前需补的稳健性证据。

### #43 A Skeleton-Enhanced AI Tutor for SQL（Pustulka, Kaiser）— T1

- **研究问题**：〔领域问题〕如何帮学生学 SQL 的 semantics（语义）而非仅 vocabulary+syntax——SQL 是声明式语言（说"要什么"不说"怎么做"），与过程式语言差异大，商学院数据库课中 SQL 仅占约 3 周，需在有限时间内掌握语义。〔技术问题〕如何在游戏化练习中给个性化、简短、快速、低成本的 AI 提示而不直接泄露完整答案、不打断游戏心流（flow），并自动评估提示质量。**核心思路**：提出 SQL Skeleton（骨架）——给学生由 keywords + placeholders 组成的"骨架"（比喻博物馆恐龙骨架，学生需填出"活的、语义正确的 SQL"），叠加 LLM 生成的片段式 hint，并用 LLM-as-judge 评估反馈质量。
- **主要方法**：**框架**：游戏 SQL Scrolls（当前 250 个 query，材料分 books/tasks）承载练习；学生提交 attempt 后连同正确答案发给 LLM，返回 JSON 格式的 hint + explanation。**关键细节**：架构跑在 Docker，前端 GUI，query 在浏览器内用 SQLite 评测，后端存 specification/正确答案/数据/期望结果/test cases，新增 skeleton 字段，底层 MongoDB 记录每个 query/结果/提交时间。Skeleton 由脚本一次性静态生成（无 AI 参与），按 task 难度激活/停用（easy task 停用以免反推答案）。Prompt 设计："you are an SQL teacher" + 正确期望 query + 学生输入 + 要求 JSON 输出 + 规则（展示与正确 query 偏差的 <15 字符正确片段；解释 <20 词）。评估框架（LLM-as-judge）：构建两条 workflow（Google 生成 hint / OpenAI 生成 hint），用 OpenAI evaluation framework + 自定义 pedagogical feedback grader 打 1–7 分、pass 阈值 5；测试集 30 个 = 10 SQL mistakes + 10 progression cases + 5 easy queries + 5 Parsons problems。用户研究采 crossover design，14 个任务（7 有 AI / 7 无 AI），问卷基于 24 名参与者。
- **核心贡献**：① 提出 SQL Skeleton 这一 scaffolding 原语——以"keywords+placeholders 的骨架 + 片段式 hint"针对性训练语义而非语法，并可按难度自适应激活/停用；② 系统量化"质量 vs 延迟"硬权衡并给出工程结论：100% 质量需 14 秒（破坏心流不可用），light thinking 为 4.7 秒/97%，no thinking 为 80% 质量且 <1 秒、仍可比肩 GPT-4，low/high temperature 无差异——选"thinking 关闭"的方案以 <1 秒换足够质量；③ 以 LLM-as-judge（自定义 pedagogical feedback grader，1–7 分阈值 5）自动评估 hint 质量并核算成本（30 个 tests 约 $2 / 60,000 tokens），把反馈质量评估工程化。
- **研究启示**：**future work**：计划下学期投入生产；坦承未实现"只显示一半 skeleton"之类细粒度 scaffolding 控制。**产品/落地启示**：实时教育 LLM 产品中"延迟是一等约束"——宁可牺牲推理深度换 <1 秒，以保护交互心流；"给 hint 不给答案 + 默认 3 次失败后才出 AI 按钮（可配置）"是对抗 AI 依赖/认知惰性的可复用 guardrail。报告暴露两个未决争议供产品方警惕：用户研究显示短期任务完成量上升（无 AI 平均 2 个、有 AI 平均 4 个）但学生对长期 retention 信心最低、长期学习效果作者明言 "unclear"；以及评分基于 SQL 执行结果而非语句本身，小数据库下错误 SQL 可能返回相同结果（提问者主张用非 LLM 的 solver 作 baseline 以不漏 error，双方未达成一致）。

### #47 When Can We Trust Early Warnings? Leakage-Excluded Early Outcome Prediction（Le, Abel, Laforge）— T4

- **研究问题**：〔领域问题〕LMS（Moodle 等）产生海量交互日志被广泛用于"早期预警、识别挣扎学生"，但很多早期预警模型的早期数字**不现实**——pipeline 在不经意间"从未来泄漏信息（temporal leakage）"，导致 early performance 被严重高估，进而误导何时、对谁干预。何时才能真正信任一个早期预警？〔技术问题〕如何把"时间有效性（temporal validity）"强制进特征工程与训练流程，使每个观测时刻 t 的预测只用 t 之前合法可得的数据，并量化"若允许泄漏，早期 AUC 会被夸大多少、夸大来自何处"。**核心思路**：提出的是"一套协议而非一个模型"——以 temporal validity enforcer 为核心的审计协议：先在时刻 t 截断、再查泄漏、只用合法数据提取特征训练，并系统枚举常见泄漏陷阱及对策。
- **主要方法**：**框架**：原始日志（content / quiz / forum / assessment）→ 时间表过滤器在不同 cut-off（3/7/14/21… 天，按周 week1–week8 评估）截断 → 泄漏检查 → 特征提取 → 训练。**关键细节**：明确两个研究问题——① 随观测到更多课程进度，预测准确率如何增长？② 若允许后续信息泄漏，早期数字会被高估多少、来自何处？枚举 **4 类常见陷阱及对策**：generation-before-radiation 式时序错置、与未来 assessment 表关联的 leakage schedule、在 split 前做标准化/预处理（应只在训练集 fit 再应用到测试集）、score-view（成绩观测后才可见）——对策含"每个 cut-off 在任何聚合/变换前切""预处理只在训练集学""除非显式可观测假设否则排除 score-view"。数据集：OULAD（Open University Learning Analytics Dataset），约 22,000 学生 / 22 门课程运行。基线模型约 10 个（logistic regression、random forest、KNN、gradient boosting、Naive Bayes 等），固定参数与随机种子、80/20 train/val，指标 ROC-AUC、AP（average precision）。
- **核心贡献**：① 提出 leakage-excluded 的早期预警审计**协议**（temporal validity enforcer + "先截断再查泄漏"），把"早期预警可信吗"从直觉问题变为可操作的方法论；② 通过泄漏模拟量化高估幅度——允许泄漏时 early ROC-AUC 从约 0.72 飙到约 0.96（完全不现实），且增益主要来自 assessment 数据，正是该协议要防的；③ 系统归纳 4 类泄漏陷阱及对应防护，并实证早期阶段必须依赖 engagement signals（点击 / 活动），因成绩尚不可得。其余发现：准确率随时间稳定提升、约 week3 出现最大跳变；tree 类早期数据稀疏时随机森林最佳、后期 gradient boosting 接管，Naive Bayes 持续最弱。
- **研究启示**：**future work**：尝试 LLM / transformer / 时间序列模型；跨平台部署（Moodle 等）；加入隐私保护机制。**产品/落地启示**：对任何做"早期预警 / at-risk 检测"的教育 AI 产品，本协议是一个可直接复用的工程纪律——上线前必须做按时间切分的泄漏审计，否则离线 AUC 可能虚高至不可用；报告亦明确两条边界——"活动量 ≠ 学生成功"（有学生少量活动也学得好），且 OULAD 未记录材料发布时间，须显式做"成绩可得性假设"，产品在各校自有 LMS 落地时需结合隐私保护与本地学习轨迹重新校准。

### #82 Agentic Tutoring for Visually Impaired Students: Concept Graphs and Audio-Tactile Learning（Kawulok, Maćkowska, Maćkowski）— T1

- **研究问题**：〔领域问题〕如何教视障（blind / visually impaired）学生计算机科学，重点是算法学习与画流程图（flowcharts）——现有方案（如 Quorum、Blocks4All、Swift Playgrounds accessibility）多基于 VoiceOver，且普遍"先为明眼人设计再适配盲人"，并非从一开始就以盲 / 视障为核心。〔技术问题〕如何用一套从底层为视障设计的 audio-tactile 机制承载触摸交互，并用 concept graph + 错误建模驱动一个能决定推进或复习的自适应 tutor。**核心思路**：以自研多年的 adaptive audio-tactile 机制（触觉打印物叠在可感应触摸的平板上、触摸即朗读音频描述）消除"图形与盲文描述需分别打印多张纸"的痛点，并把 CS 课程分解为 concept graph，配 knowledge / error 双向量驱动 agentic tutor。
- **主要方法**：**框架**：三组件 audio-tactile 系统——(A) web 平台（教师创建可打印图像并为图形元素配替代描述）；(B) 平板应用（即使有打印物覆盖也能感应触摸，触摸某元素即朗读其音频描述）；(C) 触觉打印物叠在平板上。**关键细节**：本研究把计算机科学课程分解为 14 个 concept nodes（concept graph，每节点有后继，需掌握前一个才能进下一个），每个 concept node 配一个 mistake node 细分错误类型；两个向量——knowledge vector + error vector（演讲中口误把两者都说成 knowledge vector）——驱动 tutor 决定推进或复习。评估为后使用问卷：system use / information quality / interface quality / overall（Likert）得分 2.4 / 3.16 / 2.40 / 2.70，接近文献参考值；SWAT 主观工作负荷 demands 3.7（中等）、resources 4.7、understanding 最高、overall 平均 4.6（积极）；NASA-TLX 相比仅用纸质打印的旧方法，所有类别工作负荷低约 30 分。面向 primary / lower-secondary 层级；数学课程已先验证（40–43 个 concept nodes 覆盖整个大纲），CS 是首次试验。
- **核心贡献**：① 一套"从底层为视障设计"而非事后适配的 adaptive audio-tactile 教学机制（触觉打印物 + 可感应触摸平板 + 音频描述），消除旧方法图形与盲文分纸打印的负担；② 用 concept graph（14 节点）+ 每节点配套 mistake node 的双向量（knowledge / error）建模，把无障碍交互与自适应推进 / 复习决策结合为 agentic tutor；③ 实证可用性——问卷得分接近文献参考、NASA-TLX 工作负荷较纸质旧法降约 30 分，说明学生学习更省力。
- **研究启示**：**future work**：扩展到化学、生物等图形密集学科；根据学生错误动态生成描述；让 agentic tutor 自适应调整课程路径。**产品/落地启示**：无障碍教育 AI 产品应把"以目标障碍群体为核心设计"作为出发点而非合规补丁——本案的触摸感应 + 音频描述 + 触觉打印物组合，提供了图形密集学科（CS / 数学）面向盲生的可复用范式；报告中明确局限——时间有限，问卷只测"使用"而非"成绩（grading）"，尚未做正式学习成效研究，产品化前需补学习增益证据与跨学科 concept graph 的可扩展性验证。

### #97 Cognitive and Contextual Risk Assessment for Single-Pilot Operations: A Companion Agent（Mazouzi, Frasson, Ben Abdessalem）— T8

- **研究问题**：〔领域问题〕未来驾驶舱将走向单飞行员操作，飞行员压力 / 事件负荷随之增大，需要 agent 辅助其遵守程序。核心痛点：throttle reduction（推力下降）在地面有相同遥测特征——可能是正确的拒绝起飞（rejected takeoff），也可能是危险的引擎失效；不知道飞行员当前执行到哪个程序步骤就无法区分。〔技术问题〕如何在飞行员不愿戴 EEG 飞行的约束下测量其认知状态，如何用本体 + 模拟数据训练相位分类器以提供程序上下文，并用多 agent 架构把"看大脑 + 看程序"的结论安全地转为决策建议。**核心思路**：以 companion agent 融合飞行员认知状态（EEG / 面部热成像）与程序步骤遥测，经相位分类器给出飞行相位上下文，再传给 recommender / deviation agent 做风险评估与建议。
- **主要方法**：**框架**：多 agent 架构 = coordinator + 两层（input layer + safety layer），safety layer 强制 I/O 契约 + throttling 防止 agent 淹没 coordinator；companion agent（看飞行员大脑 + 程序）→ recommender agent（决策：通知塔台 / 接管飞机），另有 deviation agent 检测注意力缺失（如机组人员进舱分散注意，非飞行员之过）。**关键细节**：认知状态测量用 EEG（最快、约 1 秒内反应）+ 心率 + 眼动，关键发现"飞行员工作负荷↑时鼻子温度↓"（EEG 与面部温度相关），故可用仪表盘热成像相机替代 EEG（规避工会对戴 EEG 飞行的抵触）。用 OWL（Web Ontology Language）写 37 个任务的有向无环图（WOL 本体），每任务带前置条件 / 状态约束 / 控制动作；数据由 Perfect Pilot Executor 在 X-Plane 12 模拟器按本体执行，但用人类反应时附近的随机化时序产生真实飞行员式变异，跨多操作条件（飞机重量、风速风向、时段、云量、rotation aggressiveness）共 1,748 次运行，按 EASA 飞行数据监测标准的 8 项质量准则评估 → 1,464 nominal + 284 deviant。相位分类器用 temporal convolution network（TCN）输出 8 个飞行相位的概率分布，融合 EEG + 程序步骤遥测通道。
- **核心贡献**：① 用面部热成像（鼻温）作为 EEG 的可部署替代来间接测认知 / 工作负荷，解决"飞行中无法戴 EEG"的现实约束；② 用 OWL 本体 + X-Plane 12 的 Perfect Pilot Executor 生成带真实变异的标注数据集，并用 TCN 相位分类器做相位识别（其中一个发现是：仅用 nominal runs 训练却能在 deviant runs 上保持准确，提示模型学到底层相位结构而非记忆正常模式）；③ companion / recommender / deviation 分工 + safety layer 的多 agent 设计，用相位上下文消解程序遥测的歧义。具体数值（runs 数、准确率等）以正式论文为准。
- **研究启示**：**future work**：核心缺口是需真人飞行员验证（目前为模拟飞行员行为），计划从 Air Canada 招志愿者、由工业伙伴严格遵循程序规则；下一步检测飞行员疲劳（alpha 波 = 疲劳信号，即使知道程序也无法执行）并向副驾 / 塔台发信号。**产品/落地启示**：高风险 / 安全攸关场景的 agentic 辅助系统，提供了"生理状态感知 + 程序上下文 + 多 agent 安全分层"的可迁移范式——其中"用可部署的间接生理信号（热像）替代不可接受的侵入式传感（EEG）""区分风险评估（问题多大）与求解（找方案）、优先看人的状态"两条原则，对教育 AI 中的认知 / 情绪状态感知与干预时机设计同样有启发；报告中明确当前仅模拟飞行员、真人验证缺失，是落地前的关键空白。

### #102 MuTSE: A Human-in-the-Loop Multi-Use Text Simplification Evaluator（Roșcan, Petre, Dumitran, Dumitran）— T7

- **研究问题**：〔领域问题〕LLM 可生成针对特定 CEFR 等级（A1–C2）的简化文本，但跨模型 / 跨 prompt 的比较与优化极复杂；现有工具二分割裂——开发者用无可视化的命令行脚本，教育者被困在 chat 界面，而 BLEU / BERTScore 等标准指标缺乏教育者需要的细粒度可解释输出。〔技术问题〕如何在单一可视化环境里并行比较多模型 × 多 prompt 的简化结果，如何用稳健的语义对齐避免标准余弦相似度的假阳性（把简化句错链到语义相似但结构错误的原句），如何让教育者自定义评分标准并叠加自动可读性指标。**核心思路**：构建人在环（human-in-the-loop）的多用途文本简化评估器，以并行比较工作流 + 交互式语义对齐引擎 + 可定制评分框架，弥合"原始 LLM 生成"与"定性人工评估"之间的鸿沟。
- **主要方法**：**框架**：三大贡献——① 并行比较工作流（多 prompt × 多模型并发运行）；② 交互式语义对齐引擎；③ 可定制评分 + 自动可读性指标框架。**关键细节**：解耦异步 client-server 架构，后端 Python + FastAPI + Together AI（serverless LLM 访问），前端 Vue.js 3，NLP 管线用 spaCy + sentence-transformers + scikit-learn。并发用 Python asyncio（无需多进程 / 线程），3 prompts × 4 models = 12 个请求同时发，延迟由最慢单任务界定而非求和。三层对齐策略解决余弦相似度假阳性：BERT embedding → TF-IDF fallback → positional metric 兜底，并加 linearity bias / positional penalty 强制结构顺序，纯 CPU 运行无需 GPU。开放权重模型（llama、qwen 等）零配置接入。agnostic 评分：教育者自定义标准（词汇适当性、语法准确、教学适用性），系统算加权总质量百分比 + 实时指标（Flesch-Kincaid grade、压缩比），并高亮 LLM hallucination 与伪影。
- **核心贡献**：① 把"原始 LLM 生成"与"定性人工评估"之间的鸿沟用单一可视化的人在环工具弥合，面向教育者 / 研究者；② 交互式三层语义对齐引擎（BERT → TF-IDF → positional + positional penalty），系统性消除标准余弦相似度把简化句错链到结构错误原句的假阳性，且纯 CPU 可跑；③ asyncio 并行比较工作流使多模型 × 多 prompt 的延迟由最慢单任务界定，配合 agnostic 可定制评分 + 自动可读性指标，给教育者细粒度可解释输出而非单一黑箱分数。
- **研究启示**：**future work**（作者自述 3 项）：缺真实课堂的正式用户研究（SUS 问卷）；缺对人类参考的定量对齐验证；需扩展到跨语言简化。**产品/落地启示**：面向教育者的 LLM 评估 / 比较类产品，"并行多模型对比 + 可解释细粒度指标 + 教育者自定义评分 + 纯 CPU 部署"是降低使用门槛与算力成本的可借鉴组合；报告暴露的边界值得产品方警惕——小语种（如罗马尼亚语）数据不足导致语义层简化质量差，句子合并在句法层（hypotaxis→parataxis）可行但词汇语义层有问题，且可读性指标只看词长 / 句长、未涉及主题与篇章连贯等多层次特征，这些是简化质量评估走向生产前需补的能力。

### #100 Proposing CAiT: a Socio-Technological Framework for AI Literacy in Early Childhood Education and its Implementation in Tangible Augmented Reality（Hofer, Roetzer）— T5

- **研究问题**：〔领域问题〕4–8 岁儿童日益暴露于 voice assistants/smart toys/recommendation systems 等模仿人类行为的 AI，却对底层机制零理解；该年龄是认知/情感/社会/文化发展的敏感窗口（pretend play/role playing/imitation 是 empathy/theory of mind/moral agency 的基础），而现有 AI 素养框架多从 8 或 10 岁起步、且"entirely screen-based"。〔技术问题〕RQ1 如何在早教阶段 operationalize 并有效教授 AI 素养（识别可衡量的 criteria/dimensions）；RQ2 有了 operationalization 后如何设计可测量、可测试有效性的教学环境。**核心思路**：以 CAiT 五维社会技术框架补齐文献中 5 个结构性 gap，并用 tangible augmented reality 把抽象 AI 机制变为可物理操纵、可观察的具身学习，嵌入家庭与照护（care）情境的非正式学习。
- **主要方法**：**框架**：CAiT 五维架构（对应 5 个 gap）——① Epistemic（AI 素养内容本身，用 AR overlays 渲染 data flows、可视化模型输出使机制 observable）；② Developmental & cognitive（受年龄/发展阶段约束，age-graded concept selection，pre-operational→concrete-operational 分阶段）；③ Institutional & regulatory（governance-aware deployment、one-device computation）；④ Pedagogical & interactional（caregiver/school participation、companion interfaces、context-sensitive scaffolding prompts）；⑤ Material & technological（embodied interactions，如带 AR markers 的 colored data blocks）；后两维通过 scaffolding interaction 整合。**关键细节**：理论部分为 critical integrative review——初筛 300+ 篇、最终纳入 14 篇，按 type/learning domains/ethical considerations/family&peers/embodied learning/developmental grading/institutional context/explainable AI 等多维度编码。AI literacy 定义为使个体能批判性评估 AI、有效与 AI 协作、把 AI 作为有目的工具部署的 competencies。课程设计三层对齐 OECD/European Commission/UNESCO 能力框架（Understanding AI mechanisms / Responsible & risk-aware usage / Safe & effective integration），分龄 4–5 岁 vs 6–8 岁；三大教学原则——故事与游戏的非正式整合（story-based game）、4E 认知（Embodied/Embedded/Extended/Enacted）、experiential & social learning（与父母/照护者的 intergenerational 学习）；实证部分为已规划的 three-phase design-based study（mixed methods，含与儿童及家长的 co-creation workshops，现场跳过）。
- **核心贡献**：① 提出面向 4–8 岁、嵌入 caregiver 与 tangible/embodied 学习的 CAiT 五维社会技术框架，系统化应对文献中 5 个被忽视的 gap（embodied 维度缺失、tangible+AR 模态缺失、caregiver 仅被当 peripheral、4–8 岁段未被整合、institutional/regulatory context 极少建模）；② 把抽象 AI 机制落到 tangible AR 的具体实现（colored data blocks + AR markers 可视化 data flow、one-device computation），对抗早教 AI 素养"全屏幕化"与儿童实际学习方式（操纵环境/想象/pretend play）的错位；③ 给出对齐 OECD/EC/UNESCO 的 age-graded 课程与 4E 认知教学原则，将 AI 素养类比为应与数感/语言发展同步形成的读写素养。

### #71 Generating Program Tracing Learning Problems by Converting Program Code（Shashkov, Bartenev, Molchanov, Sychev, Anikin）— T1

- **研究问题**：〔领域问题〕program tracing 是 CS 教育少数基础技能之一（连接语法与"代码如何执行"的语义理解），入门编程大班、不及格率高，mastery 训练需要大题库的高质量 tracing 题、手工不可行。〔技术问题〕如何全自动生成 fully valid & usable 的学习题、无需人工过滤，让 ITS 能按需自增题库（automatic loop），并把前作（仅 expression-level）扩展到更大代码片段（statement-level / 控制流结构）。**核心思路**：放弃 LLM 直接出题（因常产 unsolvable/过难/违反语法、需人工过滤），改用确定性 deterministic pipeline——从真实代码经语义中间表示生成可验证、可跨语言的 tracing 题。
- **主要方法**：**框架**：确定性 pipeline——挖掘 GitHub 仓库→解析为 "Meaning Tree（语义树）"（先 parse 为 concrete syntax tree，再转为 language-agnostic 抽象 Meaning Tree 捕获语义，据此跨语言生成等价代码 Python/C++/Java）→改进 parser 升级到 statement-level→构建 Control Flow Graph + runtime 模拟 trace 路径/保留变量状态/验证循环终止性→过滤→附 pedagogical metadata。**关键细节**：Meaning Tree 关键设计——assignment 在某些语言是 expression、另一些是 statement，故设为"既非 expression 也非 statement、转换时两者皆可"的特殊概念，并对 loops/branching 做语言无关分类；循环分三类（迭代次数不可预测 / 可预测且易在代码中修改 / 可预测但不易修改，后者需 rejection 以免生成错题）。过滤丢弃语法错误/不完整/malformed/含语言依赖不可翻译（如 Java 的 Scanner）/歧义行为/非终止循环。metadata 含 usage concepts/required skills/difficulty/nesting depth/cyclomatic complexity，技能定义基于 "thought process graphs" 理论 + 学科领域模型自动化。评估：数据集为 CodeWorkout（10 万+ Java 新手程序，但质量极差——仅 <2% 可成功编译），最终仅 119 个程序通过过滤、生成 218 道 tracing 题（Python/C++/Java，唯 nesting depth 普遍较浅）；课堂研究 N=18（10 名 Volgograd State Technical University 软工本科生 + 8 名某私立软工学校学生，约 14–16 岁），pre-test→ITS 内做 30 道生成题→post-test+问卷，测 absolute/relative learning gain，用 Wilcoxon signed-rank（验 pre/post 差异）+ Friedman test（比较题型类别提升模式）。
- **核心贡献**：① 提出 Meaning Tree 语义中间表示 + CFG/runtime 终止性分析的确定性全自动出题 pipeline，可跨语言（Py/C++/Java）生成等价 tracing 题且无需人工过滤，使 ITS 能 automatic loop 自增题库；② 明确以"可验证的确定性生成"对抗"LLM 直接出题"的不可信（unsolvable/需人工过滤），是"可信自动出题"路线的代表；③ 课堂实证 pre→post learning gains 多为正且统计显著，Friedman test 显示各类别提升模式一致（学生真正改善了对原理的理解、甚至略提理论理解，而非只背"在 tutor 里怎么做"）。
- **研究启示**：**future work**：挖更多源码/入门数据集、自动修复语法错误代码（如补分号）、改进 termination analysis、更大规模课堂研究、支持 recursion/exceptions；其中 recursion 是真正难点——非递归可只显示"某函数被调用"，递归必须显示每次调用传入的确切参数以区分多次未结束的同名调用（已做首个实验证明递归编码可行），而生成递归题暂更适合 template-based。**产品/落地启示**：确定性可验证 pipeline 适合需要"题目一定有解、trace 与可见代码一致（否则系统失信）"的自动出题产品，且设计为与 LMS（如 Moodle）耦合、已 GitHub 开源、几乎无独立 web 界面。报告暴露的现实约束：教育源代码数据集质量极差（<2% 可编译）严重限制产量；不支持 recursion/external iterators/复杂数据依赖；用户唯一较强负面反馈是"meaningful trace 步骤过多耗时"；课堂样本太小（N=18）。

- **研究启示**：**future work**：完成正式测试取得实质效果数据；提出用 AR 或 earpiece（单独语音直送 tutor 耳朵、不进会议）避免对话停顿；扩展到个性化 agent、多语言、更广学科样本。**产品/落地启示**：对教育 AI/ITS 产品而言，本案例把"延迟"和"介入时机"暴露为一等约束——报告明言发数据到拿 hint 之间的 delay "可显著削弱提示的时效"，且 tutor 收到 prompt 会停下来读造成对话里的 odd pause；产品化需把"何时推、推多深、如何不打断"内建为机制（按轻重缓急排序、在对话停顿/讲完时才推）。隐私接受度是现实落地障碍（学生不愿被录的"红灯"心理），匿名化+录音不持久化是可复用的合规设计；"AI 促进人与人之间的互动"而非替代是其核心立场。

### #48 Detecting Wandering States in Novice Programming through an LLM-Based Diagnostic Oracle（Kim, Lee）— T1

- **研究问题**：〔领域问题〕入门编程课中学生从语法到逻辑各种报错、及时反馈重要但教师没时间；LLM 修代码近乎完美但直接给答案会剥夺学生思考、造成 over-dependency。〔技术问题〕把 AI 教育焦点从"how to fix code"转向"when to intervene"——区分 productive struggle（有效挣扎，不该打断）与 unproductive stagnation / wandering state（无效徘徊，正该帮的时机）；wandering state 定义为学生反复改代码但毫无实质进展（源于 conceptual gap），持续会 frustrated 直至 drop out。RQ1 能否早期检测 wandering 起点（用 WLD 指标）；RQ2 能否分类 wandering 类型（用 MBI 指标）。**核心思路**：重新定义 LLM 角色——从 simple answer provider 转为 diagnostic oracle，分离有效挣扎与无效徘徊。
- **主要方法**：**框架**：四阶段——①抽取初始失败代码+修订历史；②用 AST（abstract syntax tree）比较代码结构找 logical deviation；③对每次提交算 WLD 诊断是否 wandering；④用 MBI 分类类型。**关键细节**：LLM as Diagnostic Oracle——学生首次提交时 LLM 找 logic gap 并设定 target node count，作为固定基线（"fixed GPS concept"，不每次重评）。WLD = Weighted (cumulative) Logical Deviation，是"逻辑里程表"度量浪费的努力、命中阈值即标记 wandering，complexity cap 防单个大错误过早触发。MBI 度量学生在命中阈值前改代码的"强度"（只分析阈值点之前历史以去噪）：MBI 高=frantic trial-and-error，MBI 低=完全 frozen（概念阻塞）。AST 用 innermost rule（parent 与 child 都错时丢冗余 parent 只留具体 child node 保证 target count 精确）、用 unique fingerprints 只算纯逻辑意图、忽略注释/空格风格改动。五种 persona：Stable、Strategic struggle（不越阈值、自己能解、不帮）；越阈值后分 Logic（完全 frozen）、Mindless tinkering（疯狂乱猜="shotgun debugging"）、General wandering（居中）。数据集：220 条 code trajectories / 60 名学生 / C 语言课，排除首次即成功的；诊断用实验使用的 Gemini 3 Pro，prompt 要求做"minimal structure-preserving changes"。
- **核心贡献**：① 提出"LLM as diagnostic oracle + 固定 target node count"范式，把 LLM 从答案提供者重定向为只判断"后续编辑靠近还是偏离解"的诊断器，将焦点从 how-to-fix 转向 when-to-intervene；② 设计 WLD（累积加权逻辑偏差，含 complexity cap）+ MBI（阈值前修订强度）两指标及五-persona 分类体系，可早期检测 wandering 起点并区分 frozen / tinkering / general 三型；③ 提供经验定标证据——LLM 诊断与人类专家达成 100% agreement、complexity cap=5（~85% 任务只需 ≤5 个 target node）、WLD 阈值=3（修订次数标准差在 2→3 出现拐点），并给出实证分布（31% straight struggler 成功 withhold 干预、23% general wandering 被准确捕获）。
- **研究启示**：**future work**：扩大数据；引入 severity-aware weighting（当前所有节点等权，未来给关键逻辑流更高权重、次要结构问题更低权重）；下学期做扩展研究。**产品/落地启示**：对编程 ITS 产品而言，"检测无效徘徊的起点再介入"是对抗 LLM 直接给答案造成 over-dependency 的可落地干预策略，且 frozen 学生需概念讲解、tinkering 学生需逻辑流追踪，提示按 wandering 类型分流不同帮助形态；阈值/cap 依赖经验数据定标（拐点法）是其方法可迁移性的关键，产品化前需在目标人群重新校准。报告亦提示其潜在风险——整套诊断依赖 LLM 可靠性，本文用 100% 人工一致性背书，跨语言/跨课程的稳健性待验证。

### #23 Understanding Perceived Assessment Difficulty: A Multimodal Analysis of Behavioral and Structural Factors（Lamgarraj, Joiron, Parant, Dequen）— T4

- **研究问题**：〔领域问题〕现有数字评估系统从聚合成功率估题难度（如 70% 学生答错→判"难"），但①没说明"为什么难"②忽略学习者真实认知体验（选择题可能靠猜对）③结构因素与行为因素几乎总被孤立研究④预测模型完全依赖历史作答数据、新题部署前无法估难度。〔技术问题〕如何在 design time（设计时）就估难度而无需历史数据，并解释哪些因素重要。**核心思路**：以学生本人 1–7 量表显式自评感知难度作 ground truth（而非从成功率反推），融合眼动+鼠标行为信号与题目 2×2 结构因素做多模态难度建模——"难度是题目结构与学习者行为交互涌现的现象（emergent phenomenon）"。本研究属更大研究 DELTA。
- **主要方法**：**框架**：双实验共享同一 ground truth（每题后学生 1–7 自评难度）。**关键细节**：DELTA-Lab（受控眼动实验）60 名本科生，实验使用 Tobii Pro 眼动 + Python PyGaze + OpenSesame 平台，屏幕 1920×1080 划 AOI（1 prompt + 4 选项 + 1 validation），抽三类特征——眼动（fixation count/duration per AOI、saccade count、blink rate）、鼠标（area under curve 路径弯曲度、max absolute deviation、cursor velocity）、结构。DELTA-Web（在线结构实验）聚焦题目结构属性、已收 >100 可用 session 且持续在线收集、人群更广。题库三领域 psychology / computer science / general culture（CS 与 psych 对应学生背景，general culture 中立做 validation，分离 expertise effect 与 structural effect）。2×2 析因设计：prompt(short/long) × distractor(simple/complex，即语义远/近的选项)，变体 A=short+simple 最易、D=long+complex 最难；每人 36 题 counter-balanced、Python 脚本自动生成并随机化题序/选项位置。机器学习：三分类（easy/medium/difficult，按 0–3/3–5/5–7 切分），测 Random Forest、XGBoost、1D CNN，目标兼顾预测与解释。
- **核心贡献**：① 用学生主观 1–7 自评而非成功率反推作 ground truth，并首个同时融合 eye-tracking + mouse-tracking + 结构因素的难度预测框架；② 模型可在部署前（design time）估难度、无需历史数据，破解新题冷启动；③ 实证支撑"难度是结构×行为交互涌现"的中心论点——五个假设全部确认：complex prompt 使 fixation count 近乎翻倍（16→32），moderate cursor speed（~500–1000）成功率最高（~70%）而过快（>2000）掉到 50%（在猜），complex distractor 单独就把成功率压到 0.5（语义接近的选项 proximity 是支配性因素），且超过 ~100 秒后表现不再提升（结构复杂度无法靠多花时间补偿），RF top features = hesitation/area under curve/max deviation/cursor speed + prompt type & distractor，主观难度任务上达 ~68% accuracy。
- **研究启示**：**future work**：扩到真实课堂、收更多数据；用校内 LMS 的 LLM 做 item generation 与自适应评估生成；Q&A 暴露跨题型/跨领域泛化性待验证。**产品/落地启示**：对评估类教育 AI 产品而言，"设计时估难度（无历史数据）"直接解决题库冷启动与新题上线难题；多模态行为信号（眼动+鼠标）提示行为正成为难度/状态推断的可用输入，但眼动设备依赖（Tobii Pro）限制规模化，鼠标轨迹特征更易在普通 Web 端采集、更具落地性。"语义接近的干扰项是难度支配因素""超过约 100 秒表现不再提升"是可直接用于出题与限时设计的经验规律。

### #22 DEBSIM: Productive Friction for Learning Transfer in LLM-Based Argumentation Tutoring（Kim, Park, Song, Suh）— T1

- **研究问题**：〔领域问题〕LLM 能按需生成完整论证，那么当系统能直接产出成品时，如何判断学习者是在真正发展技能、还是只是在与成品交互——核心问题即"LLM 辅导中的 productive friction 能否支持 learning transfer"。两个理论缺口：方法论缺口（现有系统多测"带工具时的 in-system performance"，不测工具移除后学习者能做什么）；assistance dilemma（系统该给多少、留多少信息——给太多会 short-circuit 产生学习的认知努力，给太少学习者卡住，而 LLM 能跳过中间步骤直接出成品，把这个 tradeoff 推到极端）。〔技术问题〕如何把"该自动化什么 / 该留给学习者什么"操作化为系统机制，并用"系统不在场"的迁移导向协议证明学习而非表现。**核心思路**：提出 Productive Friction 设计理念——区分 mechanical operations（系统自动化）与 cognitive operations（保留给学习者），通过三个强制摩擦点逼出认知所有权，并以 transfer-based evaluation（评估全程无系统访问）作为 LLM 辅导学习声明的 baseline standard。
- **主要方法**：**框架**：DEBSIM 三准备阶段、导航非线性（可随时返回任意阶段）——Stage 1 Argument formation（用户选 topic + stance，系统用 Toulmin model 的 6 组件 Claim / Ground / Warrant / Backing / Qualifier / Rebuttal 自动生成结构化论证 + 3 个 claim + speech outline；第一摩擦点 = 学习者须 review / modify / extend 每张卡片才能继续）；Stage 2 Evidence enhancement（系统按论证结构推荐搜索关键词、返回排序结果并解释相关性；第二摩擦点 = 证据非自动插入，须显式选源 + 遇 verification prompt，评估 / 整合留给学习者）；Stage 3 Counterargument simulation（AI 充当 devil's advocate 交互挑战论证；第三摩擦点 = 须自己打字作答而非选选项，常触发返回前序阶段形成 autoloop）。**关键细节**：within-subject 设计，20 名大学生，每人准备两个议题（UBI / 议会议员人数），一个用 DEBSIM、另一个用 self-directed baseline（web search + notes + ChatGPT，ecologically valid）；准备 20 分钟 → 现场 Zoom 辩论 15 分钟（辩论中无系统访问）→ 30 分钟半结构化访谈；顺序用 2×2 Latin square 反平衡。分析：交互日志记录 stage transition 与 edit；opening statement 按 6 个 Toulmin 组件双标注（Cohen's kappa = 0.82）；辩论用 Walton's dialogue framework 把每个 turn 标为 challenge / clarification / confirmation / concession；后测 12 项 7-point Likert，用 Benjamini-Hochberg FDR 校正。
- **核心贡献**：① 提出 transfer-based evaluation 方法论（学习声明的证据应来自系统不在场的设定，评估任务 live debate 全程无系统访问），并建议作为 LLM 辅导研究的 baseline standard；② Productive Friction 设计策略 + 选择性自动化（mechanical vs cognitive operations 的可操作区分），通过三摩擦点把认知努力放回学习者；③ 多阶段知识组件支架架构（以 Toulmin 6 组件为独立支架与评估单元，inner loop = 论证结构↔证据整合、outer loop = 对抗反馈触发修订）。关键结果：DEBSIM 条件下 opening 论证组件显著更丰富且增益不均（Backing 增长近 4 倍、Qualifier 近 2 倍，正是新手最缺、分别对应 Stage 2 / Stage 3）；Concession 从对照的 3 次跳到 14 次且转为策略性让步；后测 12 项中 5 项校正后显著（overall debate quality 差异最大），delivery 与 general comprehension 无差异 → 指向特定差异而非整体 response bias。
- **研究启示**：**future work**：作者多次自陈样本小（n=20）、within-subject 设计无法隔离单个 stage 贡献、未单独验证 counterargument 质量及其因果，并把 over-reliance 视为最想解决的问题，指向 adaptive scaffold fading。**产品/落地启示**：对话式 LLM 学习产品应把"刻意不自动化认知步骤（productive friction / interface resistance）"作为对抗过度依赖的一等设计范式，并用"系统移除后的迁移表现"而非带工具表现来证明价值；报告暴露的尖锐争议——对抗式矛盾（counterargument / "learning by disturbing"）的效果因学习者水平而异，好学生受益、弱学生可能被不成比例地打击——提示产品需按学习者画像自适应调节摩擦 / 支架强度，避免一刀切。

### #89 GRAFHIA: A Narrative Gamified LLM Model for Graphic Design Students（Becerra, Triviño, Manrique）— T3

- **研究问题**：〔领域问题〕教育 AI 的价值不止在生成答案质量，更在于"随时间组织交互 / 学术决策的顺序（better sequencing of academic decisions over time）"。应用于平面设计本科生的毕业论文 / 选题决策——把宽泛兴趣转化为可行学术项目（topic selection、project delimitation），这类任务 ill-defined、ambiguous、progressive，需 guidance / institutional alignment / thematic convergence，且有情感维度（不确定、过载、对导师依赖、求验证）。〔技术问题〕ITS（自适应反馈）/ Human-AI interaction（对话信任、任务特异性）/ Gamification（动机进度参与）三领域通常各自孤立——对话 AI 流畅但 reactive，游戏化常简化为点数 / 榜单，ITS 常只关注 correctness；如何把对话智能与叙事进度整合为学术决策的结构化机制。**核心思路**：把 LLM 重构为"叙事内嵌的决策结构化 agent（narrative-embedded decision-structuring agent）"，在对话引擎之上叠加叙事游戏化交互核心与学术定位层，从 exploration 引导到 convergence。
- **主要方法**：**框架**：方法论为 Design-Based Research（2022–2025，渐进式精炼、非单一原型），四阶段顺序演进（明确不应被解读为单一合并样本）——① 课堂叙事游戏化（无 AI）：课程设计成 RPG 进度系统（里程碑 / 角色 / reward cards / symbolic scores），46 名学生跨 3 学期；② 对话 AI + RAG：从文档检索经验证的机构信息再生成、对齐学术标准，2023 quasi-experimental baseline 38 名学生（对照 19 / 处理 19，处理组用 LLM + RAG + vector DB 注入学术文档）；③ 探索性测试：2024，20 名学生，把样本游戏化元素并入对话助手；④ GRAFHIA 整合：叙事游戏化 + 对话 AI，感知数据 10 名学生。**关键细节**：分层架构——底层 conversational engine = LLM + RAG + institutional grounding；上层 narrative gamified interaction core（chapters / entity framing / symbolic economy / level progression 组织学生轨迹）；academic positioning layer（reflective artifacts / thematic conversion / 学术决策形式化）。交互逻辑从 exploration → convergence（早期助手问开放问题、连接兴趣与方法论后果，高级阶段变 directive 支持问题定义与可行性分析，increases structure while preserving student agency）。当前实现通过 ChatGPT 内 customized GPT 配置 + 结构化 system-level instructions，因平台 context window / 字符上限把叙事模块化为 chapters；在研版用 full-fledged AI agents 替代 custom GPTs（技术栈 LangChain / LangGraph + RAG + Azure OpenAI）。
- **核心贡献**：① 把 LLM 重构为 narrative-embedded decision-structuring agent，证明教育 AI 的价值"不仅在答案质量，也在随时间组织交互 / 学术决策"——将对话智能、叙事进度、学术定位三者整合为结构化机制（弥合 ITS / Human-AI interaction / Gamification 各自孤立的缺口）；② 难得克制的能力自我定位——明确声明当前版本不是 fully adaptive ITS，而是 "sequential academic mediator（顺序学术中介）"；③ 提供 DBR 四阶段演进的多源证据（感知 n=10：约 90% 报告课程主题理解提升、项目质量感知均值约 4.1/5、80% 愿推荐，但感知成绩提升仅中等均值 3.4——学生不把它当"成绩助推器"而当理解 / 结构化 / 决策支持；n=38 baseline 偏向处理组于 clarity / engagement / confidence）。
- **研究启示**：**future work**：缺学生产出的客观评估、更长追踪、advisor workload 指标、跨机构复现；需把 GRAFHIA 与"对话式 ITS""非叙事 LLM 助手"对比，纳入 project quality trajectories / completion time / advisor workloads 等客观指标；架构正从 custom GPT 迁往 LangGraph + Azure OpenAI 的 full agent。**产品/落地启示**：面向 ill-defined 学术 / 创作决策（非简单问答）的教育产品，"把决策序列化为 chapters / levels 的叙事进度 + RAG 机构接地 + 随阶段从开放问到 directive"是可借鉴的编排范式，且这类产品宜定位为"决策 / 理解支持"而非"成绩放大器"；报告暴露的工程约束——custom GPT 的 context window / 字符上限迫使叙事模块化、早期游戏化靠教师手动管理不可扩展——是从 prompt 工程迈向 agent 编排的现实驱动。

### #98 A Hybrid Cognitive–Agentic AI Architecture with a Coordinator for Virtual Pilot Assistance（Bonyad, Ben Abdessalem, Frasson）— T8

- **研究问题**：〔领域问题〕飞行员需同时控制多目标 + 应对大量噪杂告警；若加一个普通 AI assistant（额外 dashboard + 乱响声音）只会更干扰。在 safety-critical 场景，AI 推荐"何时给、给不给"至关重要——不能像普通 assistant 那样未经授权就发消息，需先聚合 + 安全边界 + 授权再发，且每个决策必须可追溯（traceable）。〔技术问题〕普通 agentic AI 没有 safety boundary、不遵守 policy，orchestration 只能编排 agent 却不保证安全；如何把 policy / governance / 可追溯性嵌入一个能仲裁多 agent 冲突推荐的协调架构，并兼顾深度分析与实时性。**核心思路**：做"深度分析但慢的 primary/brain 架构"与"快但浅的 reactive 架构"的混合（hybrid），以 Coordinator 为唯一中枢强制 governance——所有感知输入与 agent 消息必先过 coordinator 再分发 / 回流，安全优先级高于一切。
- **主要方法**：**框架**：混合 cognitive–agentic 架构 = 左侧 primary/brain（深度、复杂、慢）+ 右侧 reactive（快、浅）的 mix，Coordinator 为核心。**关键细节**：所有 agent 的消息与多模态感知输入（飞行员的 motor / profile、camera、eye-tracking、voice、face 等）必须先过 coordinator，由其分发给 agents、结果经其回流；内部用 blackboard + 分层 control layers，agent 间用 snapshot protocol 通信，提到 deliberation agent。关键要求：安全攸关情况系统须进一步审慎；控制权初始可不经 coordinator 直接交飞行员（紧急 takeover）；常规下 agent 不能直接发消息给飞行员，必须全部发给 coordinator；每个决策须可追溯、含 policy proposal + final decision。评估配置含约 6 个指标（reproducibility 等，转录破碎）。第一阶段离线评估显示 full configuration（含 governance / coordinator）表现最稳定一致，去掉 governor / controller 后 policy 会失控、缺 access control / decision policy；coordinator 提供快速决策且满足 governance 要求。
- **核心贡献**：① 提出以 Coordinator 为唯一中枢、强制把所有 agent 决策约束到 governance 的混合 cognitive–agentic 架构，直击"普通 agentic AI 无 safety boundary / 不遵守 policy"的痛点；② 把 policy / 安全优先规则内嵌进架构与 coordinator，使其能据预设规则实时仲裁 agent 间的冲突建议（具体仲裁示例以正式论文为准）；③ 强约束的安全设计——agent 不得绕过 coordinator 直接联系飞行员、每个决策含 policy proposal + final decision 且可追溯，离线评估验证含 governance 的 full configuration 最稳定。
- **研究启示**：**future work**（自述）：当前仅用于有限场景；EEG 等其他模态、飞行后其他控制场景未覆盖；未来要做 compliance、实时性等；核心挑战是"如何在控制中嵌入 agentic identity"，问题不只在技术而在系统 + 服务。**产品/落地启示**：任何多 agent 教育 / 决策产品在引入自治 agent 时，本案提供了"中枢 coordinator + 内嵌 policy / governance + 预设优先级 + 全决策可追溯"的安全编排范式——当多个 agent 给出冲突建议时，需要一个能按规则仲裁、且不让任何 agent 绕过治理直接触达用户的中枢，而非单纯的消息总线；本篇仅在离线有限场景验证，具体指标与标题以 proceedings 正文为准。

### #85 Situating Early Fraction Learning in a Contextualized, Gamified and Scaffolded Environment: A Supermarket-Themed Educational Game（Shang, Wang, Wang, Sun）— T3

- **研究问题**：〔领域问题〕低龄儿童（5.5–8 岁）早期分数学习的核心问题不是"游戏有没有用"或"如何延长 engagement"，而是"如何设计能更好支持早期分数学习的分数游戏"。三大学习困难：①whole number bias（把整数逻辑误套到分数，如认为 1/8 > 1/4 因 8>4）；②multiple representations（分数可是 part-whole / ratio / measure / division-operator，对小孩像彼此割裂、难迁移）；③high cognitive load（同时想分子分母、比单位分数、在多表征间迁移）及由此引发的情绪困难，而幼儿 self-regulatory skills 尚未发育。〔技术问题〕现有数字分数游戏不足——任务多为 decontextualized 孤立 quiz、points/badges 是表面游戏化（只奖励完成不维持学习）、假设儿童能自主调节（把注意力/策略管理压给幼儿）。**核心思路**：以"四模式渐进 + 三大支持架构（情境化叙事 / 生态化游戏化 / 可选家长 scaffolding）"对应三大困难，构建超市主题分数学习游戏。
- **主要方法**：**框架**：四模式渐进设计（具体→符号推理）——①Cut Cake（按订单切蛋糕 1/2、1/3、1/4，part-whole 连续整体分割）；②Packing（按订单往盒子装相对量，离散集合分割，从单位分数到复杂分数）；③Restocking（用具体物或符号补货补满货架）；④Checkout（用乘法/小数/折扣算价格，符号推理综合）。**关键细节**：三大支持架构——Contextual embedding（嵌入"超市/店主"叙事，从空店开始经营、赚 coin 买货架）；Ecological gamification（coin 不只是奖励——前两个简单模式常开、后两个锁定，需用 coin 买货架来解锁 restocking/checkout，使游戏化与进度真正挂钩）；Social scaffolding（左上 help 按钮的可选家长支持，家长面板按 cognitive/emotional/technical 困难场景给不同策略，介入与否、用给定还是自创策略均由家长决定）。更新版已集成 AI 自适应学习 + RAG-directed LLM 支持家长 scaffolding。评估为 pilot/feasibility：community sampling 招 20 对亲子（parent-child dyads），儿童 5.5–8 岁、多数未正式学过分数，平均游戏时长 25 分钟、含前后测对话/问卷、共 33 个任务；基于对 132 名中国家长的前测（期望每周 1–3 次、每次不超 30 分钟）设计时长。
- **核心贡献**：① 把"情境化叙事 / 生态化游戏化 / 社会化 scaffolding"三大支持架构系统对应分数学习三大困难，并以四模式具体→符号渐进路径串联多表征；② 提出 ecological gamification——用 coin 解锁后两个模式，使游戏化奖励与学习进度内生绑定，区别于 points/badges 的表面游戏化；③ 引入可选家长 scaffolding（含 cognitive/emotional/technical 分场景策略 + 更新版 RAG-directed LLM 后端），以家长介入弥补幼儿自调节能力不足，并经 pilot 证明三架构可成功整合、家长满意度平均 >5.5 且尤其满意 scaffolding 功能（给了介入孩子学习的"缓冲区"）。
- **研究启示**：**future work**：更大样本、直接测学习成效、纵向研究、对照试验（contextualized vs 非、scaffolded vs autonomous）；奖励频率与情绪自调节/屏幕时间的平衡留待纵向研究。**产品/落地启示**：对低龄教育游戏产品而言，"游戏化与学习进度内生绑定（解锁机制）"和"把家长作为可选 scaffolding 角色而非把自调节压给幼儿"是两条可复用设计原则；基于家长前测（每周 1–3 次、每次≤30 分钟）反推游戏时长/任务量，体现以使用预期驱动设计。报告中明确其局限——仅可行性研究、样本极小、仅依赖家长感知与观察、未测任何 learning outcomes，产品方需警惕家长满意度不等于学习成效；且"奖励频率与情绪稳定的平衡"尚无证据。

### #76 Scratching the Surface of Quantum: Context-Aware AI Tutoring for Quantum Computing（Choa, Koo）— T1

- **研究问题**：〔领域问题〕量子计算走向主流、各国已在写 K12 量子教育标准，但给儿童教量子难，三大障碍——概念反直觉（superposition / entanglement 无日常类比）、现有工具不适合儿童（Qiskit / Cirq 需文本编程，IBM Quantum Composer 等可视化工具无教学脚手架）、支持资源稀缺（少教师资源、少适龄材料、卡住时无即时个性化帮助）。核心动机是"lack of in-the-moment help"。〔技术问题〕如何把 block-based 编程与 LLM 智能 tutor 结合用于量子计算教育（声明"据我们所知尚无此结合"），并让 tutor 的回应接地于学习者实时工作区（电路）状态而非仅靠文本输入。两个研究问题：RQ1 学生搭电路时浮现哪些 interaction patterns？RQ2 学生和教师实际如何体验该系统？**核心思路**：构建 Scratch Quantum——扩展 block-based 量子编程环境 + context-aware LLM tutor，每轮把电路状态解析成结构化对象与自然语言摘要注入 prompt，使 tutor 成为嵌入任务的真实学习伙伴。
- **主要方法**：**框架**：系统四组件——前端 Scratch 环境 + 量子 blocks（捕获 block sequence 与 circuit state，量子 blocks 分 circuit construction / execution & measurement / visualization 三类，可与 loops / conditionals 无缝组合）；后端 Qiskit Aer simulator；LLM tutoring engine；instructor dashboard + data layer。**关键细节**：context-aware tutor 机制——学生发问时系统评估 circuit state（已执行则取存储的 circuit objects，否则解析画布 block tree → 组装为结构化对象：有序操作 / 参数 / 最新测量结果），与近期 chat history 一起每轮注入 prompt；prompt 三支柱 = system prompt（persona + 适龄语言）/ circuit context（画布 blocks 翻成自然语言摘要）/ conversation history（最近 10 轮）。LLM 用 Claude Sonnet，输出硬上限 200 tokens（保证解释简短适龄）。Tutor 四原则写进 prompt：不给完整答案（只给 hint + 下一步）、用熟悉类比、始终引用电路、用引导性提问促思考。评估：20 名六年级学生（11–12 岁，无量子经验但约 1 年 block 编程基础），workshop = 3 节 40 分钟课；10 名 >10 年经验在职教师；数据为 tutor interaction logs（prompt + AI 回应 + 精确 circuit state + 时间戳）+ UEQ-S 问卷（pragmatic / hedonic 两维度）；两研究者把 prompt 编入 6 类（Cohen's κ = 0.82），对 46 个 session transcripts 定性回顾归纳 4 种 interaction patterns。
- **核心贡献**：① 据称首个把 block-based 编程 + context-aware LLM tutor 结合用于 K12 量子计算教育的系统（Scratch Quantum，已提供在线 web-based simulator）；② context-aware 机制——把学习者实时电路状态解析为结构化对象 + 自然语言摘要注入 prompt，实现"circuit-aware error correction"等只有上下文感知才能做到的精确诊断（如学生设 2 个 qubit 却把门作用到不存在的 qubit 6/7，tutor 坚定指出）；③ 提炼 4 种 interaction patterns（A ideal progression 26% / D concept check 37% 最频繁 / C circuit-aware error correction 21% 作者最看重 / B persistent difficulty 16% 暴露"缺自适应策略切换"局限）与 3 条 K12 量子 tutoring 设计原则（最小化认知负荷、反馈接地于实时工作区状态、加入自适应脚手架）。关键数据：20 人中 19 人用 tutor，211 次交互 / 46 sessions，on-topic 约 80%，55% 交互发生在学生正搭电路时；UEQ-S 出现学生 vs 教师分歧（学生 hedonic 好但 pragmatic 低于平均，教师各维度 excellent 进 top 10%，归因于数字素养差异而非量子概念难）。
- **研究启示**：**future work**：实现自适应脚手架机制（检测挫败、动态切换教学法）是最关键下一步；迭代界面补 pragmatic 差距；做更大更多样队列的纵向研究。**产品/落地启示**：LLM tutor 产品的关键差异化在于"context-aware / grounded"——把回应接地于学习者实时工作区状态（而非仅文本聊天）是产生有效教学行为的根源；对 LLM 幻觉的三层防护（prompt 层限定主题 + 生成层硬上限 200 tokens + 课堂层 instructor dashboard 实时监控 / 删改的 human-in-the-loop）是可直接复用的工程实践；报告中明确两处未决——tutor 当前未做学生分层建模（虽已测知识水平 + 前后测、计划写期刊）、学生持续卡住时只会重复（pattern B），是 K12 落地前需补的自适应与个性化能力。

### #36 Scaling Deep Knowledge Tracing through Distributed Model Training（Morland, Dewan, Lin）— T4

- **研究问题**：〔领域问题〕随教育数据集增长，如何扩展（scale）自适应学习中用的模型——研究处于 AI in Education × Deep Learning × Distributed Systems 交叉点，偏实践 / 工程侧，不研究模型本身精度，而研究"如何高效训练并部署"。场景：教育机构（尤其预算有限者）想用 DKT / 自适应学习系统，但深度学习训练 + 推理算力成本高，限制了可部署性（deployability）。〔技术问题〕DKT 用序列深度网络建模学生对知识组件的掌握度（本质是分类，预测下一题对 / 错）；如何用分布式训练把不同规模的 DKT 模型高效铺到多 GPU / 多节点，并量化"加 GPU 是否掉点 / 收益如何随模型与数据规模变化"。**核心思路**：用 DDP（Distributed Data Parallel）+ NCCL + Ring All-Reduce 的标准分布式栈训练 4 个 DKT 模型，跨多种计算范式对比算力成本 vs 部署效率，给 practitioner 按"模型规模 × 数据集"选 GPU 数的实践曲线。
- **主要方法**：**框架**：测试 4 个 DKT 模型——DKT（原始 LSTM-based）、DKT+（加正则化）、SAKT（Self-Attentive KT，自注意力层）、AKT（Attention-based KT，对 data embedding 加额外正则化）。**关键细节**：分布式训练策略 DDP（各 GPU 复制相同模型 + 优化器、喂不同数据、同步梯度）；数据分发用 Distributed Sampling（随机非重叠切分，兼具减偏作用）；通信后端 NCCL（CUDA 优化，支持 InfiniBand / PCIe / NVLink）+ C10D rendezvous 做节点间协调；同步算法 Ring All-Reduce（各节点前向→反向算梯度→环状分发→各自优化步，可在 backward pass 期间就开始通信减少 GPU idle）；框架 PyTorch + torchrun（支持 checkpointing 容错）。计算范式对比 single-node CPU / single-node single-GPU / single-node multi-GPU / multi-node multi-GPU。硬件 2× AWS EC2 G5.24xlarge（每台 4× NVIDIA A10G，共 8 GPU）；数据集 ASSISTments 2009 / 2012 / 2015；指标 Epoch time、AUC、GPU hours（= GPU 数 × 训练时长，折算成本）、Speedup、Parallel Efficiency。
- **核心贡献**：① 把 DKT 训练的"算力成本 vs 部署效率"系统量化——大模型（AKT、SAKT）随 GPU 数增加 epoch time 显著下降、speedup 与并行效率最高，小模型（DKT、DKT+）几乎无收益甚至因通信开销略变慢（diminishing returns），给出 practitioner 按"模型 × 数据集规模"选 GPU 数的实践指导；② 关键正面结论——跨所有 GPU 配置模型精度（AUC）保持一致，即"加 GPU 不掉点"，大模型在高 GPU 数下"同精度、更高效率"；③ 给出面向预算有限机构的可部署性工程路径（DDP + NCCL + Ring All-Reduce + PyTorch/torchrun），明确把研究焦点从"刷精度"转向"高效训练与部署"。
- **研究启示**：**future work**：更大 multi-node 集群 + 作业编排（Slurm）；更大更新更多样数据集；评估更先进架构（transformer-based / graph-enhanced / attention-augmented / hybrid）；增强 model performance（不只是 speed）。自承精度偏低——用了相对基础的训练 + 极少超参优化、AKT 实测性能比文献低，且只测了同构实例、异构未验证。**产品/落地启示**：教育 AI 工程正从"模型精度竞赛"转向"工程落地"，本案对预算有限机构提供了直接可用的成本-效率决策依据（大模型才值得上多 GPU，小模型上多 GPU 反被通信开销拖累）；报告中明确的边界——异构硬件 + 大规模集群未验证、模型 / 数据集偏小、超参优化不足——是产品级规模化前需补的空白。Q&A 还揭示动态增删 GPU 的实践：Ring All-Reduce 收敛一致性 + 节点数校验机制理论上可处理 GPU dropout 而不中断，但会有性能下降。

### #33 From Classroom Narratives to Structured Pedagogical Events: A Cold-Start Study with LLMs（Bian, Zhai, Lu）— T7

- **研究问题**：〔领域问题〕ITS 正进入真实物理课堂（人-AI 协作大趋势），但原始课堂文本长、乱、无结构，计算机无法直接理解。目标是把无结构叙事文本→清晰结构化行为数据，使 ITS 能量化统计教学策略、在时间线上给教师自适应反馈。核心约束是在 cold-start 低资源条件下做自动化抽取——cold-start 困境有二：数据稀缺（标注教学行为需深度专家知识，普通人无法标，高质量数据严重不足）、领域差异（不同学科课堂文本差异大，为新领域标新数据极贵）。〔技术问题〕两个 RQ——RQ1 极端数据约束下哪种抽取范式最有效（传统规则 / 有监督小模型 / LLM 三选一）？RQ2 不同 prompting 策略（zero-shot vs CoT）与模型容量 / 规模如何影响结果（是否"更大模型 + 复杂 prompt 总更好"）？**核心思路**：把任务定义为 Pedagogical Event Extraction（把叙事映射为结构化行为三元组 type / trigger / arguments，称"课堂行为的 DNA"），用统一框架对比 rule-based / discriminative / generative 三条 pipeline，并系统研究 prompting × 模型容量的交互。
- **主要方法**：**框架**：层级事件 Schema = 28 个细粒度行为，整合 FIAS 框架（interaction patterns）+ iSAP 框架（cognitive depth），两大类按"谁主导"分（Teacher-led：knowledge presentation〔含 lecturing / summarizing / evaluation / demo explanation〕、interaction、initiation 等；Student：如 collaboration〔含 group discussion / presentation / exhibition〕），每个行为配自然语言定义让 LLM 无需训练数据即可理解。**关键细节**：三条 pipeline——Rule-based（500 关键词词典 + 固定规则匹配）；Discriminative（BERT，传统有监督，需训练数据）；Generative（Qwen 系列，测 zero-shot / few-shot / CoT，两种规模：小 7B 与大 Qwen-Max）。数据集构建：原料为真实 K12 教学视频（数学 / 科学 / 语言多学科），教育专家看视频分段写结构化叙事描述，用生成式语言模型做 style optimizer 清洗 / 统一风格，人工复核后一致性 κ = 0.80。评估极严：一个预测得分须同时满足行为 type 与专家标签完全一致 + trigger 找到确切词（文本 overlap 超阈值，转录为 ">18%" 疑为更高阈值的转录误），对模糊猜测零容忍；指标 Precision / Recall / F1；三种 prompting = zero-shot（只给 schema + 定义）/ few-shot（3 个示例）/ CoT（先写推理步骤再答）。
- **核心贡献**：① 把课堂叙事→结构化教学行为定义为 Pedagogical Event Extraction（type + trigger + arguments 三元组）+ 28 类整合 FIAS / iSAP 的层级 schema（每类配自然语言定义），并构建经专家标注 / κ=0.80 的 cold-start 数据集；② 实证 LLM 是 cold-start 教学分析的最佳选择——Qwen zero-shot F1=0.70，比 BERT（0.40）与 rule-based（0.39）高近 13 分且未用一个训练样本，且 BERT 随样本 16→216 增加 F1 很快变平且始终够不着 Qwen zero-shot 虚线，印证"深度语义理解 >> 表层模式匹配"；③ 揭示 prompting 有效性高度依赖模型容量的 reasoning-perception trade-off——小 7B 模型 CoT（0.62）反而低于 zero-shot（0.70，因写推理步骤引入小错误 / 幻觉、对简单文本 over-thinking 虚构不存在的 quiz event），大 Qwen-Max CoT 最佳（0.74，全研究最高），凝练为"prompt 复杂度必须匹配模型规模"。
- **研究启示**：**future work**：主要只测 Qwen 系列，需扩展到更多 SOTA 模型验证普适性；pipeline 只处理文本叙事，真实课堂是多模态，未来整合原始 audio / video 信号；优化 pipeline 速度以实现真实课堂实时反馈回路；Q&A 透露正用 multi-agent collaboration 进一步提升精度。**产品/落地启示**：低资源 / 标注稀缺的教育 NLP 任务（如把课堂转录变成可量化教学行为），LLM zero-shot 应作为默认强基线而非先标数据训小模型；"prompt 复杂度须匹配模型规模"是可量化的工程原则（小模型用 zero-shot、大模型才上 CoT），可直接指导产品的模型 / prompt 选型；报告中明确多模态与实时低延迟反馈尚未完成，是从文本原型走向真实课堂产品的关键工程缺口。

### #46 A Terminological Framework for Bridging Student Modelling and Learning Design（Mohseni, Asensio-Pérez, Bote-Lorenzo, Gómez-Sánchez, Dimitriadis）— T4


- **研究问题**：〔领域问题〕ITS、knowledge components (KC)、knowledge tracing、smart learning environments 等领域常用 skill / concept / KC / competency / learning objective 等术语，但各研究社区 / 应用语境下含义不一致，导致 learner model 与 instructional goal 之间的错位（misalignment）——尤其数据驱动方法自动生成的建模单元常对不上课程结构 / 教学意图。〔技术问题〕如何不创造新定义、而是综合这些术语在各领域已有用法，提供清晰术语框架，连接细粒度学习者建模与宏观教学 / 课程结构，并使其对"ITS 的主要用户——教师"可理解。**核心思路**：提出一个三层"桥梁"式术语框架（非严格层级），综合而非新创定义，连接 learner-modeling constraint（如 KC）与更宏观的教学结构（competency / learning objective）。
- **主要方法**：**框架**：三层术语框架——① Instructional & curricular level（最高层）：learning objectives（描述教育产出、指导活动与评估）；② Integrative level（中层）：competencies（更宏观能力，连接多个 skill / KC，可含 attitude / values 维度）；③ Cognitive modeling level（底层）：knowledge components (KCs)，用于学习者建模 / 追踪 / 自适应的细粒度单元，最贴近 ITS / knowledge tracing。**关键细节**：skill 与 concept 不作为独立层，而横跨多层——concept 描述领域知识 / KC 内容，skill 表示学习者能力 / 认知进展，二者同时贡献于 KC 和 competency；用一张表总结各教育术语 + 其所属研究领域。本篇为概念 / 综述性 poster，无实证模型或数据集；实践痛点定位明确——KC 等建模 constraint 虽广泛使用但难以与教师用于设计 / 组织教学的结构对接（教师以 learning objective / competency / concept / 活动 / 课程结构思考，对 KC 等术语不熟、分不清），且数据驱动方法生成的单元常与预定义课程要素不对齐。
- **核心贡献**：① 提供一个三层"桥梁"式术语框架，把细粒度学习者建模单元（KC）与中层 competency、高层 learning objective 系统连接，提升 modeling 与教学实践间的透明度与一致性；② 关键定位选择——不创造新定义而综合各领域已有用法，并把 skill / concept 处理为横跨多层而非独立层；③ 把"教师才是 ITS 主要用户"作为框架的可用性目标——可作 baseline 帮助区分不同领域对各术语的定义，使建模术语对教师可理解。
- **研究启示**：**future work**：进一步阐明 KC-based 学习者建模如何与 learning objective、competency 对齐，以支撑更连贯、自适应的 AI 驱动学习环境。**产品/落地启示**：对接"学生建模"与"学习设计 / 课程结构"的教育 AI 产品，常见失败点是自动生成的建模单元（KC）与教师的课程语言 / 教学意图对不上——本框架提示在产品中显式维护"learning objective ↔ competency ↔ KC"的术语映射层，并以教师能完全理解为设计基准；报告暴露其当前为概念 poster、实际落地与验证方式尚未成型（Q&A 仅以"已与合作教师多次开会、可作 baseline"作答），产品价值的实证仍待补。

### #39 The Cognitive Profiling Agent: Mapping Assessment Tasks to Bloom's Taxonomy Levels（Addiucci, Bellezza, Temperini）— T4

- **研究问题**：〔领域问题〕教育生态正从 system-centered（封闭、线性）转向 distributed & adaptive（AI agent 与学生 / 教师 / 机构 / 其它系统交互），agent 自治适配要求理解内容的认知结构；而手工应用 Bloom 分类法存在 bottleneck——极耗时、难规模化、主观（不同教育者会分歧）。〔技术问题〕AI 模型能否在"人类专家本身存在的自然变异范围内"自动完成认知分类（Bloom level）？如何从非结构化评估题目自动提取 Bloom 认知层级，喂给 learning analytics + 自适应辅导（intelligent scaffolding / dynamic feedback）+ 给教师可解释 dashboard，并尽量避免重度 task-specific fine-tuning。**核心思路**：用多语言句向量 embedding + 轻量线性分类器，评估"预训练 embedding 已捕获多少可迁移的认知结构"，把 AI 定位为对人类教育者的 augmentation（教师保留诠释权与教学责任）而非 replacement。
- **主要方法**：**框架**：四阶段 pipeline——① 收集原始题目文本；② 用 multilingual E5-large embedding 转语义表示；③ 多种训练策略（cross-domain transfer / partial domain adaptation / full in-domain）；④ 轻量线性分类器。**关键细节**：采用修订版 Bloom 分类法（动词式 remember / understand / apply / analyze / evaluate / create，强调动词描述可观察认知过程、更易计算形式化、更契合语义 embedding）；关键设计选择是避免重度 task-specific fine-tuning，转而评估预训练 embedding 已捕获多少语义结构（提升 scalability、降算力、便于跨域）。两个数据集：公开大数据集 2,450 道考试式题目（多领域、预训练覆盖）；学校真实语料 334 道上层中学真实课堂评估题（作者本人学校收集，含人文：历史 / 文学 / 哲学 / 语言 + 科学：数学 / 物理 / 化学 / 计算机 / 生物）。
- **核心贡献**：① 用 multilingual E5-large embedding + 轻量线性分类器自动做 Bloom 认知分类，避免重度 fine-tuning 而评估预训练 embedding 的可迁移认知信息——全文最重要发现是 partial domain adaptation（加入少量真实学校题）把 accuracy 从严格跨域的 0.44 提到 0.70，证实"语义 embedding 已捕获大量可迁移的认知信息"；② 关键误差分析——大多数错误发生在相邻认知层级（understand↔apply、apply↔analyze），模型并非随机出错而是镜像了人类专家本身的解释性模糊（专家标注 Cohen's kappa = 0.79，非完美），印证真实课堂任务的认知边界本就流动、上下文依赖；③ 把自动认知分析定位为"教学设计的反射镜"——模型成功复现学校集主导认知结构（analyze 类最多）、对高阶类（evaluate / create）保守预测最小化假阳性，并揭示学科差异（人文高度集中于高阶 / 诠释性推理，科学在 understand / apply 间更均衡），可帮教师检查评估是否认知均衡。
- **研究启示**：**future work**：扩充数据集、用更强 embedding 模型；用此框架分析教师布置作业 / 课堂任务的认知质量（教师未用 Bloom、未设计有意义评估时）。**产品/落地启示**：教育内容分析类 AI 产品宜采"augmentation 而非 replacement"立场——AI 提供可规模化的认知分析，human educator 居中保留诠释权与教学责任（human-in-the-loop ecosystem）；技术上"轻量预训练 embedding + 少量域内样本做 partial adaptation"是低成本跨域落地的可行路径，避免每域重训；报告中明确核心挑战是 domain shift（合成 / 公开数据难泛化到真实课堂，严格跨域仅 0.44）+ 高阶类样本少 + 认知边界本质模糊，产品需以"少量真实场景数据适配 + 对高阶保守预测"应对。

### #103 Process-Mining-Based Orchestration of Agentic AI for Adaptive Learning（Chemingui）— T1

- **研究问题**：〔领域问题〕自适应学习系统对"何时 / 如何干预"常反应迟钝、且 process mining 多用于事后分析而非实时控制。核心问题：基于 process-mining 的编排在多 agent 学习系统中实现"动态自治适配"的效果如何？为何不用 RL？因为 RL 优化 outcome 而非 process 结构、不天然处理序列 / 时序 / 重复步骤，且需大量试错、错误尝试会伤害学生，而本场景要求干预即时且易懂。〔技术问题〕如何把 process mining 从事后分析转为实时编排控制环，用多维偏差指标驱动专用 agent 的细粒度触发，并形成闭环。**核心思路**：用 Design Science Research 构建五层框架，以实时 process mining 检测偏差、合成 composite deviation score 触发 4 个专用 agent，使系统同时具备 process-aware + 实时 + agentic + 闭环四性质。
- **主要方法**：**框架**：五层——① Data Acquisition（clicks / screen time / quiz submission / scores 作连续事件流）；② Process Mining（process discovery / conformance checking / 改进，实时检测 deviation）；③ Behavioral Deviation Metrics（四指标）；④ Agentic AI Orchestration（按分数触发 4 个专用 agent）；⑤ Action（自适应交互生成并回注系统，agent 从历史学习）。**关键细节**：四个偏差指标——Structural Conformance（是否遵循规定活动序列，F1）、Temporal Deviation（长间隔 / 突发活动，RMSE）、Repetition Deviation（>2 次重复同一活动）、Performance Deviation（学业欠佳）→ 合成单一 composite deviation score（权重 α/β/γ/δ 求和=1）；四个 agent——Remediation（高重复 / 低绩效触发，如 quiz M3 三次 48% < 目标 70% → 重讲）、Feedback（结构偏差，如未答 quiz 先交项目 → 提示正确下一步）、Motivation（长时不活跃 → 鼓励提醒）、Adaptive Pathway（≥2 指标超阈值 → 重组路径并插入 checkpoint）。LLM 用 Qwen3（Alibaba Cloud Model Studio），prompt 工程四技法（Role / Persona / Few-shot / Chain-of-Thought）；process mining 工具用开源 ProM。数据：128 名 master 学生、项目管理课程、27,000 条 traces（属性 activity label / timestamp / module / duration / score）。
- **核心贡献**：① 把 process mining 从事后分析转为实时编排控制环，提出唯一同时 process-aware + 实时 + agentic + 闭环的五层框架（对比传统规则式 / RL / 基础 tutoring / 教育 process mining 各缺其一）；② 用多维偏差指标（structural / temporal / repetition / performance）合成 composite deviation score，实现细粒度、可解释的 agent 触发，并量化效果——Structural F1 0.67→0.82（p<0.01）、Performance correlation 0.79→0.87、学习成绩平均 +6.8%、模块完成时间降 74%、composite weighted deviation improvement index +22.85%；③ 揭示"生态级协调比单 agent 消息更重要"，并给出 agent 贡献画像（Remediation agent 贡献最大约 30%、Adaptive Pathway 单项最低但系统级影响最广）。
- **研究启示**：**future work**：引入情绪 / 认知负荷建模；概率式参考模型支持多有效路径；自适应阈值 + 协调 agent；跨机构 + 开源 LLM 部署。作者主动自承多项局限——pre/post 设计无对照组（因果无法确证、可能受 maturation / 累积效应干扰，Q&A 中明确承认时间类指标"不太可靠"）、单课程单机构泛化受限、偏差阈值固定（非数据驱动）、仅匿名 log 数据缺认知 / 情绪 / 元认知维度、参考模型假设单一最优路径、Qwen via Alibaba Cloud 致可复现性受限（chunking / 向量化 / embedding 为 blackbox）、高并发下实时 process mining 有 latency 约束。**产品/落地启示**：自适应学习产品可借鉴"实时过程感知 + 多维可解释偏差分数 → 只触发最相关 agent"的闭环编排范式，以替代"多 agent 各自为政"或"事后才分析"；但报告中明确其因果证据薄弱（无对照组 + 时间指标不可靠）、闭源云 LLM 可复现性差、假设单一最优路径，产品决策不应据此单机构 pre/post 结果外推。

### #101 Help-Seeking Profiles and Early Detection of At-Risk Students in AI-Assisted Programming: A Multi-Semester Learning Analytics Study（Eom, Kim）— T4

- **研究问题**：〔领域问题〕AI 编程反馈在真实课堂的三个被忽略问题——①公平性（谁受益/谁掉队，AI 反馈是缩小还是扩大差距）；②help-seeking 质量（"怎么用反馈"是否比"用不用"更重要）；③能否仅凭行为早期识别 at-risk 学生。〔技术问题〕在大学必修 Python 课的自动评测平台（~200 题/学期，运行时报错可点 "view error helper" 让 LLM 给 cause of error + hint，系统 prompt 明确"只给提示绝不改代码"）中，如何用纯行为日志构建可跨学期迁移的早期预警，并刻画 help-seeking 风险信号。**核心思路**：以 Nelson 的 instrumental vs executive help-seeking、Matthew effect vs Productive Failure 为理论框架，用多学期学习分析检验 AI 反馈的公平性并定位轻量早期预警的时间窗。
- **主要方法**：**框架**：抽取 18 个行为特征，测试前 K 周（K=2/4/6）；三个模型族 logistic regression / random forest / gradient boosting；5-fold CV + 跨学期迁移测试。**关键细节**：数据为东亚研究型大学三学期——2024 Spring ~3000 人（主训练集，底层 GPT-3.5-turbo）、2024 Fall 496 人、2025 Spring ~2500 新生（独立测试集，底层 GPT-4o），prompt 跨学期保持不变。两个预测目标：dependent profile membership（help-seeking 风险信号）、low achiever（final accuracy < 0.3）。（注：报告标 2024 Fall "595 与 spring 重叠"数字疑误。）
- **核心贡献**：① 揭示 dual equity pattern——Matthew effect 与 equalizer effect 共存：AI 反馈扩大绝对差距、缩小比例差距（2024 Spring 转化率 high 77% / mid 77% / low 仅 63%，low achiever 即便有 AI 也只到 62–64%），呼应"反馈在 performance gap 大时效果最好"；② 发现 debugging 中"挣扎不 productive"——立即求助者 struggle index=0、转化率 89%(2024)/95%(2025)，每多一次尝试 odds 降 ~7.9%（无 U 型），对 low achiever 效应更强，说明 debugging 只是堆叠 cognitive load、缺 productive failure 所需的结构化探索；③ 定位早期预测 sweet spot = 第 4 周（cross-cohort AUC dependent 0.71 / low achiever 0.76 均过 0.7，第 2 周太早、第 6 周太晚），并验证三类 help-seeking profiles 跨学期稳定、dependent group 并非最差。
- **研究启示**：**future work**：（自述局限即指向后续）feedback-seeking 自愿致 self-selection 偏差、只是 association 非 causation、GPT-3.5→GPT-4o 模型变更是真实 confound、单校仅 Python。**产品/落地启示**：对编程教育平台而言，"部署轻量第 4 周预警模型（仅用标准日志、可跨课迁移）+ 即时反馈入口 + tiered escalation + 实时监控 student profile"是一组可直接落地的设计建议；但报告明确暴露 actionability gap——Q&A 问"flag 之后老师具体该做什么不同"，演讲人未给明确干预方案、仅定位为"triage 工具非决策工具"，这是该工作落地的核心缺口。另一争议性结论："底层瓶颈是学生能力（student capacity）而非反馈质量"（反馈质量在成功/失败 episode 间无差异），提示产品方对"加更多 AI 反馈即能补差距"保持审慎。

### #94 Same F1, Different Feedback: Measuring Diagnostic Coherence in Programming Error Classifiers（Park, Kim）— T4

- **研究问题**：〔领域问题〕编程辅导系统通过对 compiler error 分类来选反馈，分类器处在每个反馈决策的最上游。〔技术问题〕标准 F1 把所有 misclassification 同等惩罚，但 within-domain 错误（反馈仍有用）与 cross-domain 错误（反馈类型全错）的 pedagogical 后果天差地别，无标准度量能捕捉这一点。**核心思路**：提出 taxonomy-aware 的诊断一致性度量族（DCR/PSCR），让"相同 F1 但反馈后果不同"的分类器可被区分——度量从混淆矩阵直接算、排名不依赖某超参。
- **主要方法**：**框架**：提出 DCR（Diagnostic Coherence Rate）——DCRD（错误预测是否停留同一 domain）、DCRC（是否停留同一 category）；配套 PSCR（severity score，把 domain-crossing 程度并入，越低越好，same-category 错误低 severity、cross-domain 错误=1）。**关键细节**：Taxonomy 两级——top 层 syntax / semantic 两 domain、下分 8 个 leaf class，基于编程教育文献且兼容 Java(javac) 与 C(GCC) 诊断输出（taxonomy 来自作者去年 ITS 会议早期工作）。数据集 CSEDM（美国大学 CS1 课）+ CodeNet（IBM 竞赛编程数据集），均 long-tail、按 user ID 切分；9 种 loss × 5 seeds × 2 数据集，backbone UniXcoder，10 epoch 取 best-val-F1 checkpoint，flat 与 coherence 度量对比。
- **核心贡献**：① 提出 DCRD/DCRC + PSCR 一族 taxonomy-aware 度量，可从混淆矩阵直接算、排名不依赖超参，捕捉 flat F1 掩盖的"反馈后果差异"；② 实证"相同 F1、不同 coherence"——macro-F1 仅差 2.6 点但 DCRD 跨 89.7–93.4，某两 loss F1 几乎相同但 cross-domain error rate 10.3% vs 6.73%（高 53%）且五 seed 稳健，并发现"core loss"micro-F1 居前但 DCRC 低（同 semantic 组内混淆更多）会被 flat 度量掩盖；③ 揭示 loss 选择风险由 long-tail 的 location/frequency 决定——CSEDM 上 syntax domain 仅 59 样本时 CBC 把 DCRD 拉低 3.1 点、CodeNet 同类大 35 倍则几乎不变，分布越 skewed loss 选择对 coherence 影响越大。
- **研究启示**：**future work**：（自述局限即指向后续）DCR 是混淆矩阵行为度量、未链 learning outcome；severity/超参是可解释设计选择、未对齐人工判断；仅 1 backbone / 2 数据集 / 1 taxonomy。**产品/落地启示**：对依赖错误分类来选反馈的编程辅导产品而言，"选模型不能只看 F1，要看 cross-domain error rate / 诊断一致性"是直接可用的选型准则，尤其在 long-tail 错误分布下 loss/模型选择会显著改变学生实际收到的反馈质量。报告暴露两点未决——Q&A 指出度量的是 inter-classifier coherence 而未引入 human ground truth（taxonomy 是 Java/C 优化的代理），以及"semantic 错误的跨域跳变是问题固有还是方法不成熟"是 open question；另有听众建议"先问 compiler 是否有 syntax error 再分类"可规避跨域问题（作者认可但未实现），是可落地的工程改进点。

### #56 An Agentic Adaptive Intelligent Tutoring System with Confidence-Aware Student Modeling Using Hybrid CGACell Networks（Constantin, Bălcău）— T1

- **研究问题**：〔领域问题〕在 agentic learning ecosystem 中，自治 tutor 做个性化难度调整时，从教育数据估计的学生知识水平本身含不确定性（观察活动可能不完整/有噪声/与底层知识状态仅间接相关），不应把每个预测都当同等可靠。〔技术问题〕如何用估计的可靠性（confidence）来选择下一步教学动作，避免基于弱证据做激进难度调整，并让学生建模同时具备预测准确与概率可靠（校准）。**核心思路**：confidence 居于本工作核心——置信高则按估计调整路径、置信低则先请求额外诊断证据而非贸然调难度，用 Hybrid CGA-cell 输出校准良好的完整后验分布支撑这一决策。
- **主要方法**：**框架**：Hybrid CGA-cell 三组件融合——①神经特征提取（输入 X → latent H，sigma 激活）；②ECA rule 110 cellular pooling（H 二值化为 cellular state → ECA rule 110 演化 → patch pooling 得紧凑向量）；③遗传算法优化结构超参（binarization threshold / ECA 迭代数 T / pooling size P / latent dim M / learning rate，crossover 用 ECA-110 生成的二进制掩码而非纯随机），最终 softmax 输出完整后验分布。**关键细节**：三知识级别 beginner/intermediate/advanced（分数 <60 / 60–80 / ≥80）。Confidence-aware 自适应：后验经时间平滑（系数 α 平衡响应性 vs 稳定性）得 P̃，confidence = 平滑后验最大值，低 confidence 不直接触发难度增减而是先做诊断活动（即三类别 + 第四"低置信诊断 fallback"决策）。闭环 agentic loop：观察证据 X_t → 算后验 P_t → 平滑 P̃ → 算 confidence C_t → 选动作 A_t → 累积新证据 ΔX_t → X_{t+1}=X_t+ΔX_t。数据集 OULAD（约 42,000 学生，预处理后）+ Student Performance（约 400 学生）；70% train(含 10% validation)/test、10 次独立种子；base 配置 latent~64 / 10 次 ECA 迭代 / pooling size 4 / lr 0.01，GA population 40、50 代。Baselines：SVM、Random Forest、MLP、1D-CNN、BKT、DKT；指标含 Accuracy、Macro-F1、Balanced accuracy、MCC、Kappa、Log-loss、ECE（校准尤为关键）。
- **核心贡献**：① 提出 Hybrid CGA-cell 学生建模——融合神经特征提取 + ECA rule 110 cellular pooling + 遗传算法（用 ECA-110 掩码做 crossover）优化结构超参，输出完整且校准良好的后验分布而非仅类别标签；② 把 confidence-aware 决策内建为一等机制——低置信时触发"诊断 fallback"而非贸然调难度，避免基于弱证据的激进难度跳变，是更负责任的自适应决策；③ 实证优于全部 baseline 且校准更优——OULAD accuracy 0.92 / micro-F1 0.91、Student Performance accuracy 0.89 / micro-F1 0.88，log-loss 与 ECE 均更低，各知识级别 precision/recall/F1 均衡，离线序列模拟显示更平滑轨迹 + 减少不必要 fallback。
- **研究启示**：**future work**：（自述局限即指向后续）当前为离线验证、模型不与学生实时交互、仅用现成 learning analytics 数据估计知识并模拟自适应决策。**产品/落地启示**：对自适应 ITS 产品而言，"把后验置信度当一等输入、低置信时先诊断再调难度"提供了一条对抗"基于弱证据误调难度"的可落地决策范式，且 ECE 等校准指标应与 accuracy 并列为模型验收标准（因策略依赖后验 confidence）。报告暴露核心短板——Temperini 在 Q&A 直指模型在静态数据集上评估、但真实学生会在交互中改变，离线验证不等于实时交互，产品化前需补真实学生在线验证。

### #42 X-ELRA: Deterministic Runtime Governance for Trustworthy Agentic Adaptive Learning Systems（West, Yang）— T6

- **研究问题**：〔领域问题〕随着自适应学习系统越来越自主（决定何时帮、何时收手、接下来呈现什么），人类越来越无法在 runtime 看见和解释系统行为，且这会影响学习者的 self-regulated learning。〔技术问题〕"更准的预测模型"并不能修复这种 opacity——可信运行需要的是对"系统实际做了什么"的治理（governance）而非更好的预测，且需与 XAI（SHAP/LIME 只解释模型级一个数字）区分开、回答系统行为层面"为什么这么做/想让我做什么/何时会变"。**核心思路**："probabilistic in, deterministic governance, recorded out"——在系统内设清晰边界分离"AI 提议(perception)"与"学习者实际体验(enactment)"，用确定性治理控制器把概率性 ML 输出约束为可 replay/audit 的决策。
- **主要方法**：**框架（工程问题）**：三段式——左侧 ML 组件读取学习者、产出 perceptions + candidate actions（保持概率性、可持续改进）；中间 deterministic governance controller；右侧学习者只体验通过 controller 的内容。**关键细节**：controller 先把输入 serialize 成单一 fixed ordered snapshot（冻结），同一 snapshot 永远产出同一决策+决策轨迹（可 replay/audit）；按 priority 顺序跑一组 routines，每个 routine 解析为四结果之一——skipped / blocked（预算或 cooldown 阻止）/ executed-with-action / executed-with-deliberate-non-intervention；发出单一 decision trace 作为 first-class explanation object，投射给学习者 + 存档审计。分离契约 agent-agnostic——LLM feature / generative feedback model / chatbot tutor / 自主 agent 都可放 upstream 提议 candidate actions（甚至 candidate prompts），统一经 governance controller 决定 enact 什么。实现：Python + FastAPI 服务、部署于真实 Python fundamentals 课（~192 个 resources）、浏览器内写/运行代码直接采集真实信号（code error patterns），90 名学生在用，decision trace 通过 Open Learner Model 投射；三组对照研究进行中（process-level summaries from trace / model-level explanations / no-governance），比较对 understanding/predictability/trust/cognitive load 的影响。
- **核心贡献**：① 提出 deterministic runtime governance 架构——以 serialize→fixed snapshot→priority routines→decision trace 把概率性 ML 输出约束为可 replay/audit 的确定性决策，区别于传统 rule-based ITS 只记录最终 enactment；② 把"每一个被考虑过的决策"记录为一等解释对象——含被 withheld 的帮助及原因、含 deliberate non-intervention，使治理可解释、可审计；③ 提供 agent-agnostic 的分离契约——任意 upstream（LLM/生成反馈/chatbot/自主 agent）的 candidate actions/prompts 统一经 governance controller 决定，并以 90 名学生的真实部署确立工程可行性。
- **研究启示**：**future work**：完成三组对照研究的实证（明确声明 poster、empirical 进行中）。**产品/落地启示**：对 agentic 自适应学习产品而言，"把治理层与预测层显式分离、用确定性 snapshot 保证可 replay/可审计、把含被拒帮助的 decision trace 当一等解释对象"是把"可信/可解释"从事后补丁变为架构内建的可复用范式，且 agent-agnostic 契约让其可包裹任意 LLM/自主 agent 上游。代表性观点"一个完全可解释的模型仍可能嵌在一个其整体行为无人能 account for 的系统里"精确点出与 XAI 的分工边界——产品方需在"模型可解释"之外单独建设"系统行为可治理"。

### #66 Ensuring Reliability in Programming Knowledge Tracing: A Re-evaluation（Kim, Kim）— T4

- **研究问题**：〔领域问题〕在为新建"编程作答数据集"做 benchmark 时，发现已有 Programming Knowledge Tracing（PKT）模型结果难解释、行为异常，担心这些不可靠的 benchmark 数字会被后续研究当作对比基准。〔技术问题〕三问——① implementation choices 能否虚高分数？② temporal order（时间顺序）是否改变结论？③ 在正确评估 + 充分调参后，复杂 PKT 模型是否仍优于简单模型？**核心思路**："Reliability before architecture"——先把评估协议做可靠（修正数据泄漏与调参不公），再判断复杂模型是否真有用。
- **主要方法**：**框架**：重审 PKT 模型（**DKT** baseline、**Code-DKT**(用 pass+attention 表征 code)、**EKT/ECKT**）+ 三项修正 + 受控评估协议。**关键细节**：数据集 **CodeWorkout**（约 17,000 interactions、约 400 students、5 个 assignments，沿用先前 Code-DKT/HKT benchmark 设置）。三项修正：① **Shield（注意力 look-ahead 修正）**——原实现 softmax 沿时间轴计算导致未来步信息泄漏到当前步，改为在单次 submission 内计算 attention、去除跨时间 look-ahead；② **T-line / 时间对齐修正**——原实现按"原始行顺序"而非时间戳排序 attempts，导致较晚 attempt 出现在较早位置、模型"提前看到"晚发生的 attempt → AUC 偏乐观，改为按时间戳重排；③ **Per-assignment 超参调优**——不同 model/assignment 最优超参不同，强制全局统一会使部分模型欠调。受控协议：train/test=80/20，train 内 5-fold，用一个指定 fold 搜超参、固定后训练 5 个 split → 5 个模型在同一 held-out test 上评估，报告 mean±std、固定随机种子；序列长度对比 max=50 vs 100（95 分位长度在 5 个 assignment 都 >50）。
- **核心贡献**：① 系统暴露并修正 PKT 评估中的两类数据泄漏/不公——attention 沿时间轴的 look-ahead 泄漏（Shield 后分数下降，证明原实现确实用了泄漏）与按行序而非时间戳排序（T-line 后多 assignment AUC 变化，证明原行序影响了报告结果）；② 证明"调参公平性"足以颠覆结论——仅改训练设置、不改架构，DKT 在 assignment 439 上 AUC 提升约 **0.07**，不同 assignment 最优学习率/停止 epoch（42–75）差异大、baseline 常被欠调；③ 给出"code features 并非总能提升性能"的可靠再评估——DKT 在若干 assignment（439、494）仍有竞争力甚至显著优于 Code-DKT，且 max=100 相比 50 平均提升很小且不一致（更长历史不总有用）。
- **研究启示**：**future work**：仅用单一数据集 + 有限 PKT 模型，下一步扩展到更广数据集与模型（含新建的 programming KT 数据集）；领域很专门（编程是 problem-solving 而非简单对错提交）。**产品/落地启示**：对依赖 KT 的自适应学习产品，benchmark 可靠性应先于模型复杂度——上线前需排查时间对齐与注意力泄漏、做 per-task 公平调参，否则可能误信"复杂模型更好"而过度工程化；Q&A 提出的延伸方向是把 KT 用于评估 ChatGPT/LLM 提供给学生的知识是否自洽（"看 ChatGPT 的脑袋而非学习者的脑袋"），即把 LLM-based feedback / code understanding 整合进 KT。

### #35 Proxy-Labeled Modeling of Confusion, Apathy, and Morale (Speech-Based Affect)（Chanbi, Tato）— T4


- **研究问题**：〔领域问题〕学生的"困惑（confusion）"常被课堂忽视、长期降低 engagement/motivation；多数 ITS 只能"是否困惑"二元检测，无法捕捉**强度与随时间演化**。〔技术问题〕**不存在带 CAM（Confusion/Apathy/Morale）连续标注的公开语音数据集**，专家标注成本高，如何仅用语音连续估计这三维学习者状态。**核心思路**：用 Russell circumplex 把现成情感语音库的基础情感几何映射为 CAM proxy 标签（绕过标注瓶颈），再训练深度模型从语音直接预测 CAM 连续分数。
- **主要方法**：**框架**：三步 proxy-labeling——① 收集 3 个公开情感语音库 **RAVDESS、CREMA-D、TESS**（共约 11,000 utterances，标注为 happy/sad/angry 等基础情感）；② 用 **Russell's Circumplex Model of Affect**（valence-arousal）把基础情感映射为 CAM（confusion=low valence+moderate arousal；apathy=low valence+low arousal；morale=positive valence+high arousal），论文用类别而非 0–1 连续值汇报以避免对"表演式情感语音"过度解读；③ 训练深度模型从语音预测 CAM——**prosodic features（pitch/energy/speaking rate）+ wav2vec 2.0** 表征。**关键细节**：融合策略对比 audio-only、frozen wav2vec+prosodic、fine-tuned wav2vec+prosodic、attention fusion；端到端 5-fold CV；指标 MSE/MAE/R²/CCC/Spearman；CAM 三维及"morale=well-being"的命名由工业合作方指定。
- **核心贡献**：① 提出"Russell circumplex 把基础情感映射为 CAM proxy 标签"的弱监督路径，绕过"无 CAM 标注语音数据"的瓶颈，且 proxies 锚定成熟心理学理论；② 实证最佳 fine-tuned 模型 macro **R²=0.52**、Spearman >0.71、CCC≈0.70，能正确**排序**情感状态（对自适应系统而言相对排序最重要）；③ 通过消融厘清驱动力——prosodic-only R²=0.16 → +frozen wav2vec 0.27 → 全量 fine-tune 0.52，得出"**任务特定预训练模型微调是首要驱动力、handcrafted feature 贡献有限**"，并给出三维可预测性梯度（morale 最易 R²=0.60、apathy 0.50、**confusion 最难 R²=0.47**，因同样困惑的学生声学表现可完全相反）。
- **研究启示**：**future work**：四方向——human validation（CAM 标签是理论推导、未经专家验证）、真实课堂部署（自发语音而非演员）、多模态整合（语音外还有语言/对话/行为，学生可能伪装"听懂了"）、纵向研究（CAM 随解题如何演化）。**产品/落地启示**：情感感知 ITS 可用"理论驱动的 proxy 标签 + 预训练语音模型微调"低成本起步，且应优先关注**相对排序**而非绝对值；但报告暴露的现实约束需正视——演员语音与真实学生差异大、单模态语音对个体不一致状态（confusion）预测最难、受身体状态影响难标准化，Q&A 还指出 EEG（直测情感中枢、实时）虽更准但因合作方要求非侵入式而暂列未来工作。

### #19 ConvGuide-VR: A Conversational Guidance Framework for Procedural Learning in Immersive Virtual Reality（Alghamdi, Cristea）— T5

- **研究问题**：〔领域问题〕VR 教育中"引导"常是静态/受限，而 LLM 可做自然对话；研究 gap 是无框架在 VR 中提供"基于学习理论的、自适应的、step-by-step 对话式引导"。〔技术问题〕如何把多个学习理论逐条映射到 VR 设计决策，并在 VR 中实现"区分知识提问 vs 引导请求、结合实时上下文给程序性引导"的对话 agent。场景：用沉浸式 VR 教排序算法（selection sort / bubble sort），学习者抓取/比较/重排桌上编号方块"动手"执行算法。**核心思路**：以三学习理论（CTML / CAMIL / Conversation Theory）驱动设计，构建 IVR + Context + Conversation 三模块框架，用 intent detection 把对话分流为知识问答与上下文引导。
- **主要方法**：**框架**：三模块——IVR module（虚拟环境+视觉更新）、Context module（把用户动作转结构化数据：speech/query、task performance、environment context、task state）、Conversation module（用 LLM 生成响应，关键特性 intent detection 区分 knowledge question vs guidance request、维护对话历史）；knowledge 问题用 domain knowledge data 回答、guidance 用 context module 数据回答、响应转语音播放。**关键细节**：三理论→设计映射——CTML（用音频旁白替代屏幕文字降视觉负荷、coherence 去冗余、segmentation 小步学习者控制、conversational style、redundancy 不同时文字+旁白）；CAMIL（presence 物理交互虚拟物体 + agency 自主提问/控制接收何种引导）；Conversation Theory（对话式学习、agent 充当 virtual tutor）。评测：within-subjects 受控实验、counterbalanced——Experiment 版（selection sort，自然语言提问+实时上下文引导，答 RQ1）vs Control 版（bubble sort，pre-recorded explanations，答 RQ2），两算法难度相近且互不依赖；测量四维 learning outcome(pre/post)、usability(SUS)、engagement(UES)、task performance(错误数+完成时间)；被试 32 名 Durham 大学学生、convenience sampling、多为非母语英语者、少/无 VR 经验、1 小时个体 session、£5 voucher。
- **核心贡献**：① 提出 ConvGuide-VR——首个把"基于学习理论、自适应、step-by-step 对话式引导"系统化进 VR 程序性学习的框架，填补该 gap；② 三模块架构 + intent detection 机制——区分 knowledge question 与 guidance request 并分别取 domain knowledge / 实时 context 数据作答，把 LLM 对话与 VR 实时状态绑定；③ 以 within-subjects 受控实验给出证据——learning 显著提升（p<0.05）、SUS 70.9（above-average）、task 错误极低（均 0.06），且对比 baseline 时虽 knowledge gain/usability/engagement 无显著差异但 mistakes 显著更少（更高 task accuracy 且不显著增加时间）。
- **研究启示**：**future work**：扩展到更复杂场景、改善 adaptivity 与语音交互（情绪/置信检测）、降低"不知问什么"的负荷。**产品/落地启示**：对 VR 学习产品而言，"把 CTML/CAMIL/Conversation Theory 逐条映射到设计决策（音频旁白替代文字、presence+agency、对话式 tutor）"提供了理论驱动而非拍脑袋的设计模板，intent detection 是把通用 LLM 对话约束到"知识/引导"两类可控响应的实用机制。报告暴露跨三场（#19/#38/#62）的共性痛点——学习者"不知道该问什么"带来额外 cognitive load + AI trust 问题，是对话式 VR 引导落地的关键障碍；质性反馈也显示偏好 baseline 者看重"无需 formulate question、更快、更 reliable（录播经专家校验）"，提示对话式引导需与可靠的预设内容互补。

### #13 Meta-KT: Metacognitive-Aware Knowledge Tracing with Pedagogical Optimization（Lee, Kim）— T4

- **研究问题**：〔领域问题〕知识追踪（KT）对 ITS 至关重要，但深度 KT 把学习序列当作 binary correctness 流（0/1），而 correct ≠ genuine mastery——两个 correctness 序列相同的学生（A 长响应+频繁用 hint 的 productive struggle vs B 亚秒响应+不用 hint 的 rapid guessing）被标准 binary KT 坍缩成同一 beat。〔技术问题〕已有工作把行为信号当 passive input、loss 仍同等对待每次交互 → 过拟合表面模式；如何把行为信号作为优化中的 active force，并把教学/认知理论直接写进数学优化目标。**核心思路**：把 Flavell metacognition / Vygotsky ZPD / Csikszentmihalyi Flow 三理论直接翻译成数学优化指南（而非事后分析），让模型"深听学习者如何学习，而不只是答对没有"。
- **主要方法**：**框架**：三阶段——Stage 1 算 PSC（Personalized Subjective Confidence）；Stage 2 metacognitive-aware transformer；Stage 3（核心贡献）metacognitive-aware optimization（hybrid loss）。**关键细节**：Stage 1——不用统一响应时间阈值（不同学生天然快/慢），用 Gaussian kernel 评估"个体基线反应风格 vs skill-specific 难度"，在学生预期 effort window 处峰值、在 rapid guessing/overload 处平滑衰减；结合 correctness + hint 派生 9 个指标 / 5 类（如 wrong+high PSC→overconfidence、persistent effort without hint→productive struggle、区分 strategic help vs cognitive avoidance）。Stage 2——投影 7 个特征流（skill id、response outcome、semantic embedding、9 指标等）入潜空间，gated fuser（可学习 gate 向量选择性放大 metacognitive 信号）+ additive metacognitive bias 直接加到 attention score。Stage 3——hybrid loss，由 5 类 metacognitive category 上的 gating 产出动态权重 W_meta（用 stop-gradient detach 稳定训练），自动放大高信息事件（ZPD 内 productive struggle）梯度、抑制 lucky guess/slip/cognitive avoidance 噪声。评测：ASSISTments 2017 benchmark，对比 6 个 baseline（DKT、SAKT、SAINT+ 等）。
- **核心贡献**：① 提出把 ZPD/Flow/metacognition 三理论直接翻译成优化目标的 Meta-KT——区别于把行为信号当 passive input 的已有工作，将其作为优化中的 active force；② 设计 PSC（个性化主观置信，Gaussian kernel 校准个体响应风格）+ 9 指标/5 类 + gated fuser + additive metacognitive bias + hybrid loss(W_meta) 的三阶段架构，超所有 baseline（mean AUC 0.7579、accuracy 0.6984）；③ 多级消融与教学有效性验证——移除 gated fusion 性能跌幅最大（证明简单 concatenation 不足以整合异质行为信号）、去 hybrid loss 也降 accuracy，且内部权重与认知代理高度吻合理论（productive-struggle-within-ZPD 显著正相关、overconfidence/slip/lucky-guess 显著负相关），质性案例显示模型对 low behavioral reliability 的答对维持保守 mastery 估计。
- **研究启示**：**future work**：把这些指标接 LLM，将数值 metacognitive 状态翻译成自然语言反馈；正研究面向师生的 metacognitive KT dashboard。**产品/落地启示**：对知识追踪/学情产品而言，"用个性化校准（PSC）替代统一响应时间阈值、把行为信号作为优化中的 active force、对低可靠性的答对维持保守 mastery"提供了比 binary KT 更稳健、更贴近真实掌握的学情建模思路，可直接改善 mastery 估计的可信度（避免把 lucky guess 当掌握）。报告中明确局限——仅单一 benchmark、proxies 是数学派生而非直接测量、instructional utility 未在 live tutoring 验证；Q&A 问"instructional utility 怎么落地（给反馈还是给内容）"时落地路径尚未成型，提示产品方对其教学效用需自行验证，dashboard 与 LLM 自然语言反馈是其指向的产品形态。

### #80 Designing for Reflective Engagement with Generative AI（Otero, Velander, Wallace）— T1

> 说明：注意与同名近似论文区分——本篇为 Otero 等的设计空间框架工作（格林威治大学）。

- **研究问题**：〔领域问题〕人们会用 LLM，但不清楚是否真促成学习——是真理解还是 shallow reproduction / "cut-and-paste"（论文抄袭式）；核心是**元认知（metacognition）+ agency**，要让学习者意识到"学习的实际收益在你自己、不在系统"。〔技术问题〕要在学习情境中用生成式 AI，但不知道**哪些交互机制**能真正促进学习、同时促进 critical AI thinking。**核心思路**：不直接造系统，而是用 HCI 的 **Design Space Analysis + Design Rationale**，把围绕 SRL 的关键设计维度、选项与 trade-off 显式化，使设计决策可论证、可复用，并由此推导出一个架构。

- **研究启示**：**future work**：本质是概念/方法论工作，尚无系统落地与实证证据（系统正在 finalize，pilot 待做）；计划基于该架构完成具体系统，用 mixed-methods 做 pilot studies、co-design sessions（教师/教育技术从业者共创）；如何"生成某些学习确实发生了"的证据是公开难题。**产品/落地启示**：设计生成式 AI 学习产品时，宜把 SRL/AI literacy/hybrid regulation 的 trade-off 显式写入设计文档（design rationale），而非默认即时响应——尤其 immediate vs delayed feedback、personalization vs privacy 这类张力需主动权衡；产品成功标准应从"能否通过考试"的 outcome 导向，转向"是否真促成学习/反思"，但当前教育系统的 outcome-based assessment 与 SRL 仍存张力，是落地的结构性障碍。

### #4 Hybrid Evaluation of Tutor Dialogues（Pado）— T1

- **研究问题**：〔领域问题〕教师在多个班级部署 LLM 驱动的 tutorial dialogue system 后，没人有时间每天看日志，如何在**不花大量人工**的前提下，自动且可靠地评估这些导师对话的质量——既要在开发期、也要在线上持续运行期监控 LLM 是否**事实正确**、是否**教学法对齐（didactically aligned）**（例如不直接替学生解题，而是 scaffold/鼓励）。〔技术问题〕LLM-as-a-judge 本身"有点循环"（用 LLM 评 LLM），单个 judge 约 78% 准确率、是黑盒，如何把它工程化为可信、可向学生/论文交代的评估流程，并刻意控制成本/能耗/对商业供应商的依赖。**核心思路**：提出"人类始终在驾驶座（human in the driver's seat）"的 5 步混合评估框架——用 LLM-as-judge 承接可靠类别的判定、人工只复核不可靠类别，并刻意选用 "blunt-edge"（非前沿）开源模型而非 cutting-edge 模型。
- **主要方法**：**框架**：5 步 human-in-the-loop 混合流程——① 人类先定义可接受的最低性能（类比统计检验先定 α，本例设 Mistake Identification 全流程 accuracy ≥ **95%**）；② 收集与未来实际数据尽量相似的数据集；③ 选 judge LLM；④ 逐类分析 judge 性能（accuracy-yes vs accuracy-no），找出模型"偏好类/最可靠类"；⑤ 模拟混合流程——接受 judge 在可靠类上的预测、人工复核不可靠类，据此决定是否采用/是否加复核/是否纯人工。**关键细节**：数据集为 **MRBench**（来自 ACL 2024 BEA workshop 共享任务），约 300 条数学辅导多轮对话（训练）+ ~200 条（测试），每条对话学生都犯了一个错；移除导师最后一轮回复，替换为 **7 个 LLM 导师 + 1 个人类导师**之一生成的回复 → 300 对话 × 8 数据点。4 个人工标注维度：① Mistake Identification（根任务）② Mistake Location ③ Guidance（给指导而非只说"错了"）④ Actionability（学生是否知道下一步）。3 个开源 LLM：**Mistral、Gemma、GPT-OSS**（含 120B 版本）。4 个递增复杂度 prompt：① Baseline（裸提示）② Fan et al.（共享任务文献，加 context、更 verbose）③ Role + Example（"你是经验丰富的数学导师"角色 + 列举成功/失败方式，非严格 few-shot）④ Committee of Experts（来自 Wang et al., NAACL-HLT 2024，模拟学生+数学导师+评估专家的委员会讨论）。建议每配置跑 2–3 次取稳；judge LLM 跑在公共资助的科学托管平台。
- **核心贡献**：① 提出"先设最低可接受性能阈值 + 在类真实数据上逐类模拟"的 5 步混合评估框架，把"人类对 AI 话语权重的完全控制"形式化（人类决定接受哪些类的机器判定、复核哪些类）；② 实证给出"中庸 prompt 最优"的稳健经验——最简（baseline）与最复杂（committee of experts）prompt 几乎从不最优，加一点 context/角色/示例的中等 prompt 最稳健且易迁移（Mistake Identification 上 Mistral+Fan et al. 最佳，次级任务上 GPT-OSS+Role&Example 最佳）；③ 论证 blunt-edge 开源模型路线的可行性与价值——始终优于 majority-class baseline，部分任务可比肩 cutting-edge（Guidance 可排第 3、Location 排 12/32），且单 judge（~78%）配人工复核后达 ≥95% 高质量，模拟显示 3 个 tutor 的评估可 100% 交给机器、部分可省 >90% 人工。
- **研究启示**：**future work**：作者把"在难数据子集上让 judge 也可靠"留为开放问题（不同 tutor 子集性能差异极大，必须在像真实数据的数据上模拟才能预测最终性能）；对量化模型的配置不确定，鼓励继续探索 reduced models。**产品/落地启示**：对教育 AI/ITS 产品，LLM-as-judge 不应黑盒单用，宜采用"先定阈值 → 逐类找可靠子集 → 机器+人工分工"的可交代评估范式；选用开源/blunt-edge 模型以换取可复现性、独立性与低能耗（作者强调"教今天的学生，也有义务保证 20 年后他们还有地球可住"），是降低供应商依赖与运营成本的可借鉴工程取向。

### #81 Novices' Code Comprehension Performance when Prompting LLMs for Assistance（Sajib, Chapagain, Tamang, Kendeou, Rus）— T7

- **研究问题**：〔领域问题〕随着生成式 AI 接管大量编码，**阅读/评估 AI 生成代码**成为程序员主导任务；CS1/CS2 新手用 LLM 辅助理解代码，但 LLM 会 hallucinate、新手识别不出错误，其真实教学价值需评估。〔技术问题〕新手的 prompt 特征（类型、结构、长度）如何影响其代码理解的信心、所得解释的质量、以及最终表现——三者关系尚不清楚。**核心思路**：构建受控的双阶段在线实验，把"新手自由 prompt LLM 辅助理解代码"全过程数据化（prompt 文本 + 自评信心 + LLM 解释质量 + 解题表现），分别回答 RQ1 信心、RQ2 解释质量、RQ3 表现三问。
- **主要方法**：**框架**：自建 Web 系统（前端 JavaScript、后端 Firebase、部署 Vercel），两阶段——① Pre-test（仅预测程序输出，**无 LLM**）② Main task（每题：读程序→1–5 自评理解度→写 prompt→读 LLM 解释→1–5 评有用度→写输出，**有 LLM**），每场约 1.5 小时、在线、实验员监督。**关键细节**：LLM 为 **GPT-4**（实验始于 2024 年 4–5 月，当时各 benchmark 最佳），system prompt="you are a helpful assistant... complete answer within max token length"，**temperature=0**、max tokens=1024。被试为北美两所大学 CS1/CS2 本科生 **N=38**；最终 **480 个编程响应、216 个学生 prompt、216 个 LLM 解释**，由 **3 名 PhD 研究者**按 accuracy/completeness/conciseness **二值标注**（保证一致性、减歧义）。代码 3 复杂度：Easy（loop+conditional）、Medium（recursion）、High（binary search/sort）。5 类 prompt：① Whole-program explanation（约半数学生）② Specific-logic ③ Conceptual（如"def 在 Python 是什么意思"）④ Direct output-seeking ⑤ Irrelevant。3 类长度：Long（>15 tokens）、Medium（10–15）、Short（<10）。
- **核心贡献**：① 以二值人工标注 + 统计检验，量化 prompt 类型/长度与"信心—解释质量—表现"三者的系统关系，提供新手 prompt 素养的实证证据；② 揭示"prompt 质量决定收益"的关键发现——Conceptual prompt 表现提升最大（+17pt，65→82%）、Specific-logic +15pt、信心增益也最大（Specific-logic 最高，Direct output-seeking 反而略降信心）；解释 accuracy 近完美但 conciseness 全面偏低（65%–71%）；③ 量化 prompt 长度效应——Long prompt 增益最大（71→86%），而 **Short(<10 tokens) prompt 反而使表现下降（81→79%）**，得出"用户交互策略与模型能力同等重要、结构化/概念驱动 > 直接求答案"的结论。
- **研究启示**：**future work**：prompt 训练干预、招募更大新手队列、接入最新 reasoning models、跨整学期的长期多 session 研究。**产品/落地启示**：教育 AI 产品的价值杠杆可能在"训练学生写结构化/概念性/较长 prompt"而非一味升级模型——报告记录的跨场共识是 reasoning 与非 reasoning 模型差别不大、训练策略差别更大（"明智使用 LLM 才获益，否则不会"）；且 prompt 行为本身可作为学习分析信号（prompt 变短可能反映疲劳/注意力下降，是可监测的投入度代理）。报告中明确的局限：单次 90 分钟 session 有疲劳效应、强制对每段代码都写 prompt 会产生无关/低质 prompt、样本小（N=38）泛化受限。

> 说明：本篇官方排在 Session 18，录像中在 Session 17 时段宣讲；华东师范大学。

- **研究问题**：〔领域问题〕如何把 LLM 引入高度专门的人文领域（古典钢琴，融合 declarative knowledge + 审美判断 + 艺术诠释），同时把部署成本压到真实课堂可负担。〔技术问题〕三难——① 通用模型领域知识浅（听起来合理但事实/风格常错）；② 开放式学生答案（如分析作曲家风格）难可靠评分、专家时间稀缺；③ 学校付不起大模型 API。**核心思路**：用 5 个 agent 编排教学闭环，配两条互补的轻量策略——一边用 LoRA 微调的小模型当"答题专家"、一边用 prompt 工程让通用 LLM 当"评分专家"，兼顾教学有效与经济可行。
- **主要方法**：**框架**：5 智能体——data synthesis agent（生成领域 QA）、teaching assistant agent（核心交付：托管微调模型答题 / 跑结构化评估 prompt）、student & teacher simulation agents（验证时复现真实教学场景）、performance evaluation agent（多维打分对比 baseline + 专家）。**关键细节**：**策略一（knowledge transfer / 轻量领域适配）**——4 步流水线：① **GPT-4o** 每批 100 题生成作曲家专业问题；② **Qwen2.5-72B** 生成详尽权威答案；③ 清洗 + 专家抽检（准确性/多样性）；④ LoRA 微调（base=**Qwen2.5-7B-Instruct**，rank 8 / alpha 16 / 3 epochs，2×A800 GPU 约 1 小时，**更新 <1% 参数**），合成语料 **3,119** QA pairs。**策略二（prompt engineering / 不训练）**——单一精心设计 prompt 让通用 LLM 当 expert assessor，4 组件：expert role（资深钢琴理论教授）+ 五维评分 rubric（每维数值分+文字依据）+ 结构化报告（highlights/misconceptions/improvement suggestions/recommended resources）+ 标准化 JSON 输出，基于 CoT + role-playing。评估：测试集=30 专家精选（头对头）+60 模拟学生问题+60 task-answer pairs；协议=2 名专家盲法 pairwise A/B + **GPT-4.1** 自动五维打分 + Cohen's kappa + 15% 专家抽检。
- **核心贡献**：① 提出覆盖教学闭环两端的 5 智能体框架——"LoRA 微调小模型答题 + prompt 工程通用模型评分"双轨，并以模拟师生 agent 在验证期复现真实教学场景；② 实证 7B 领域适配模型可比肩/超越大模型——盲法 pairwise 胜率 86.67%/83.33%，60 模拟学生问题五维较 base 提升 12.95% 且每维匹配或超过 GPT-4o（最大提升 +28% 在 analytical depth & originality 这一最难、教学价值最高的维度），depth/insightfulness、elaboration/richness 维度全胜 30/30；③ 验证自动评判与结构化 prompt 的可用性——自动 GPT-4.1 与专家一致性 kappa 0.74/0.66、prompted GPT-4o 加权 kappa 0.68（substantial），结构化 prompt 较基础 rubric 整体质量再升 7.79%，且 LoRA 更新 <1% 参数、本地部署、零经常性 API 成本，适合资源受限学校。
- **研究启示**：**future work**：当前模拟交互虽经专家验证但是简化的；只聚焦 declarative knowledge、未覆盖 performative skill（演奏的物理执行）→ 计划做含 audio/visual 分析的多模态框架；用真实学生、不同音乐风格列为未来工作。**产品/落地启示**：在专门人文领域，"把强大 LLM 蒸成轻量专门工具"是兼顾质量与成本的可行路径——合成 QA + 小模型 LoRA（本地部署、零 API 经常性成本）可对标大模型，prompt 工程让通用 LLM 充当结构化评分专家则覆盖"开放式答案难评"的痛点；"微调模型答题 + prompted 模型评分"的双角色分工，是资源受限教育场景值得借鉴的闭环范式。

### #84 Towards Offline Conversational Tutors: Benchmarking Quantised Local LLMs + RAG on Consumer GPUs（Mohamed, Zorec, Baksh）— T7

- **研究问题**：〔领域问题〕现有 LLM 驱动 ITS 多依赖云第三方（OpenAI/Gemini 等），带来三大问题——① 按 token 计费、成本随使用量线性增长；② 学生数据受监管、隐私管理开销大；③ 依赖性（不知哪家公司能存活、成本会否涨、需联网）。〔技术问题〕**消费级硬件（非数据中心 GPU）+ 本地量化模型 + RAG 能否替代云**，做到跑得够快、输出够好、能适配学生？**核心思路**：把云 LLM 换成设备本地运行的量化 LLM，在 5 个消费级 GPU 环境上系统基准测试不同量化方案，并加 RAG grounding 与 DPO 微调做个性化，量化"成本/隐私/可持续/离线"四重动机下的可行性。
- **主要方法**：**框架**：基础系统借鉴 "Ruffle & Riley"（LLM 生成 tutoring script 喂入对话系统，含 student agent 提问 + professor bot 帮助/评答），把云 LLM 换成本地 LLM 并加微调更新机制；基准方法基于 **MLPerf** 思路、针对消费级 GPU 制定并给出部署 overhead 指标。**关键细节**：模型为 **LLaMA-2 7B chat**（承认是旧模型，为与前人研究直接对比而选）；3 种量化——**GPTQ、AWQ、bits-and-bytes**；5 个 GPU 环境从 **TensorDock** 租用（均 16GB RAM、2 vCPU，3080 例外为 4 已调整；同 CUDA、主 Python 版本；含 RTX 2060 Super(Turing)、3080 等）；**关键坑：AWQ 库会把 transformers 降级**（兼容性），影响结果。**RAG**：用操作系统课讲义做 grounding；**DPO（Direct Preference Optimization）** 做个性化微调（给好/坏回复对引导模型）。评估：7 名修过该课的学生按 5 维度 1–5 评分。
- **核心贡献**：① 在消费级 GPU 上系统基准化"量化方案 × 架构"的吞吐/能效——给出可操作结论：**GPTQ 吞吐+能效一致最优**（最高吞吐为 Turing 环境的 GPTQ）、bits-and-bytes VRAM 最低、**AWQ 全面偏差**（但含库降级因素），2060 Super 吞吐最好却最耗能（电池设备不友好）；② 实测 **RAG 开销可忽略**——含检索+首响应 <0.3 秒、用户感知不到延迟，且配 RAG 后响应质量整体可接受；③ 验证 on-device 个性化可行——特定配置下（限 batch size 防 VRAM 不足）可在设备上做 DPO 微调（post-DPO 相似度约 4–5），并把本地 benchmark + ITS 代码开源在 GitHub；同时暴露"评分者对何为好回复分歧很大"，直接催生个性化研究。
- **研究启示**：**future work**：scope 很窄（仅 1 模型、有限硬件），未试 Gemma/DeepSeek 等其他架构、未试更适合 CPU/on-device 的 **GGUF** 量化；个性化样本小、不做泛化声明；Pado 在 Q&A 指出两点开放问题——RAG 与 LLM 能力未解耦评估（建议把 RAG 输入也呈现给评分者、单独问"英语是否正确/是否忠实 RAG"），以及 LLaMA-2 7B 易被 prompt injection（"忘记之前所有指令…"），需区分 hallucination（RAG 可缓解）与 prompt injection（需系统层 guardrails）。**产品/落地启示**：对成本敏感、隐私受限或需离线可用的 ITS 产品，本地量化 LLM+RAG 是可行替代——选 GPTQ 兼顾吞吐与能效、bits-and-bytes 省 VRAM、避开 AWQ、CPU 部署可探索 GGUF；RAG grounding 几乎零延迟成本却能把小模型输出拉到可接受；但小模型的 prompt injection 防护与"RAG/生成质量解耦评估"仍是产品化前的开放工程问题。

### #60 SoSAFE: A Multimodal Approach for Detecting Content Inappropriate for Children on TikTok（Diallo, Nakouri, Ben Abdessalem）— T6

- **研究问题**：〔领域问题〕儿童独自刷 TikTok 时会**毫无预警地从搞笑/舞蹈视频跳到危险挑战、自杀相关内容**（"每天发生在大量儿童身上"）；核心难点是**有害性不仅在视频本身**，还在音频、评论、用户互动方式，而现有方法多为 YouTube 长视频设计、只用 1–2 个模态、不适配 TikTok。〔技术问题〕① 整合社交上下文信号能否超越纯视听方法提升检测？② 自适应融合（adaptive fusion）机制是否优于静态/无策略融合？**核心思路**：把原本只含视频的审核数据集补齐为含社交信号的多模态数据集（视频+音频+文案+评论+元数据），用带 modality gating + cross-attention + BiLSTM 的自适应融合架构做 4 类内容分类。
- **主要方法**：**框架**：5 模态融合 SoSafe——video、audio、captions、comments、metadata，各有专属 encoder 部分微调（**I3D 视频、VGGish 音频、BERT 文本**），投影到共享空间后经自适应融合 → classifier 预测 4 类之一。**关键细节**：数据集 **TikHarm**（TikTok 内容审核 benchmark，**411 个视频，4 类**：Harmful/Adult/Safe/疑为 Suicide-Self-harm 类），**90/10 划分**（90% 训练+交叉验证，10% 严格隔离做最终评估）；数据增强补 3 类社交信号——① post captions（创作者文案）② comments（取评论数最高的 10 条）③ metadata（likes/shares/comment counts）。自适应融合模块 3 组件：① **Modality Gating**（聚焦每个视频的有用信息）② **Cross-Attention**（捕捉模态间交互）③ **Bidirectional LSTM + Attention Pooling**（生成统一表征）。
- **核心贡献**：① 提出面向 TikTok 短视频、整合社交上下文的 5 模态审核框架，把"评论/文案/元数据"等社交信号纳入有害内容检测；② 实测全面优于所有 baseline——隔离测试集 **Accuracy 87.13%、Micro-F1 87.4%、MCC 82.96%、AU-ROC 97.06%**（F1 比最佳 baseline 高 2.7 点），且 5 模态训练仅 5 分钟；③ 以消融研究证明"增益不仅来自加模态、更来自如何融合"——完整模型最优、**移除 comments 掉分最大**（视频本身模糊时评论帮助最大）、自适应融合 > 静态加权，提炼出"harmfulness depends not only on content but also how it is socially interpreted and received"的核心论点。
- **研究启示**：**future work**：多语言审核（multilingual moderation）、置信度感知决策（confidence-aware decision making）、可解释 AI 审核（explainable AI moderation）。**产品/落地启示**：儿童安全/内容审核类教育产品中，社交上下文（尤其 comments）是高价值且互补的信号，自适应融合优于简单加权；但报告中明确的工程/伦理约束需正视——TikHarm 数据集小（411 视频）、**评论在上传时不可得**（部署时序悖论）、社交信号有噪声/有偏，且自动审核会在敏感类别产生误报，**系统应辅助人类审核员而非取代**。

### #11 Confidence Estimation Metric for Automated Rubric-Based Essay Grading with LLMs（Solera-Monforte, Arnau-González, Arevalillo-Herráez, Arnau-Vera）— T7

> 说明：报告记录的实验任务为 rubric-based **automated short-answer grading (ASAG)**，与官方标题 "essay grading" 措辞略有出入，以报告实录为准；瓦伦西亚大学。

- **研究问题**：〔领域问题〕如何提升 LLM 驱动的自动评分（rubric-based ASAG）的准确性与可靠性，并在人工评分受限时**过滤掉低置信度评分**、把稀缺人力留给复杂作答。〔技术问题〕能否从 LLM 评分 token 的概率中导出一个有意义的置信度指标——更高的 confidence score 是否与更高的评分准确率相关？**核心思路**：在生成评分 token 时只取 rubric 允许的数值分 token，用其概率定义"预测分=概率最高的数值 token、置信度=该 winning token 的概率"，再按置信度分位验证"高置信 → 高准确率"，从而支撑人机协同过滤。
- **主要方法**：**框架**：基于数值分 token 概率的 Confidence Score + 分位过滤 + Spearman 相关验证。**关键细节**：数据集 **ASAP** corpus（约 **17,000** 开放式简答，覆盖 science 与 English language arts，10 个 distinct sets，rubric-based + 二级评分者、人类可靠性高）；Confidence Score 定义——LLM 生成评分 token 时只取 rubric 定义的数值分 token（如只可能是 1/2/3），计算这些 token 概率，predicted score=概率最高者、confidence=winning token 概率（例 token "3" 概率 0.85 → 预测 3、置信 0.85）；Prompt=standard（source material+question+grading rubric 固定 schema+few-shot 示例+强制格式对齐输出数值分与 feedback）；模型 **Mistral-7B**（因 context window 大）；评估——17,000 作答 × 10 数据集，按 confidence 分 8 个 quantiles，比较 top quantile（high-confidence group）vs 全局 baseline 均值，用 Spearman 检验高置信是否预测更高准确率。
- **核心贡献**：① 提出一个**无需额外训练、直接从数值分 token 概率导出**的 ASAG 置信度指标，把"评分可信度"工程化为可计算量；② 实证 high-confidence group 准确率显著高于均值 baseline、Spearman 相关在各数据集均可靠，验证该指标可作为人机协同过滤器（按领域 science 比 English language arts 评分更好）；③ 厘清指标性质——给出的是**相对置信度**（适合过滤不可靠结果），而非真正校准后的统计概率。
- **研究启示**：**future work**：校准该指标使其代表真实概率分布；探索其他置信度指标。**产品/落地启示**：LLM 自动评分产品可用"数值分 token 概率"低成本地估计置信度、按分位筛掉不可靠评分以节省人力，是 human-in-the-loop 评分流水线的实用组件；但报告暴露关键约束——该置信度是**相对于模型**的（Q&A 指出换模型影响最大，不同模型各有 calibration、会偏向某些分数 token，如更易输出 3 或 4），且未真正校准，产品落地需对每个模型单独标定阈值、勿当作绝对概率使用。
